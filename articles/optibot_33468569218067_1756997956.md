[L77] #### How to Set Up a Daily Event Schedule that Updates with the Passage of Time

[L78] 

[L79] With a Daily Event schedule, once an event happens, thereтАЩs no need to keep displaying the event. With OptiSync, you can configure your DataSource to ignore events which are now in the past.

[L80] 

[L81] To do this, weтАЩll need to set up an additional column in our spreadsheet. WeтАЩll call it the **Event Over?** column.

[L82] 

[L83] **![google sheets with an additional event over? column](https://support.optisigns.com/hc/article_attachments/33468600323987)**

[L84] 

[L85] In the тАЬEvent Over?тАЭ column, put this formula:

[L86] 

[L87] ```

[L88] =IF(B2<=NOW(),"Yes","No")

[L89] ```

[L90] 

[L91]   | **NOTE:** This formula will only work if both the Date and Time are present in the Event Time cell. The тАЬB2тАЭ part should match the actual event time, i.e. B3 for Zumba in the Example. |

[L92] 

[L93] B2 is your reference value, which maps up with the event time. This formula will work regardless of whether or not the current date is referenced in the тАЬEvent TimeтАЭ column.

[L94] Within the settings of your Google Sheet, please ensure Recalculation is on (e.g., On change and every hour/minute), otherwise the formula won't recalculate.

[L95] 

[L96] ![google sheets tutorial image with Calculation highlighted and red arrow pointing at On change and every minute option](https://support.optisigns.com/hc/article_attachments/33468569084819)

[L97] 

[L98] You can access this in your Google Sheet by selecting **File** **тЖТ** **Settings** **тЖТ** **Calculation.**

[L99] 

[L100] All the тАЬEvent Over?тАЭ column does is check whether or not the event time has passed with a simple yes or no. When this screenshot was taken, it was between 1:00 and 3:30, so the first event to be shown should be тАЬRowing,тАЭ followed by тАЬSurrender Yoga.тАЭ

[L101] 

[L102] To do this, weтАЩll make use of the **DataSource** **Filter**.

[L103] 

[L104] To get started, highlight the data you want to filter, then hit **Settings.** Then, hit the **Filter** option under your DataSource.

[L105] 

[L106] ![firefox_69WJbcItk0.gif](https://support.optisigns.com/hc/article_attachments/42920711657491)

[L107] 

[L108] This screen will appear:

[L109] 

[L110] ![optisigns datasource filter](https://support.optisigns.com/hc/article_attachments/33468600337171)

[L111] 

[L112] What follows is, essentially, an [AND statement](https://support.microsoft.com/en-us/office/and-function-5f19b2e8-e1df-4408-897a-ce285a19e9d9) or an [OR statement](https://support.microsoft.com/en-us/office/and-function-5f19b2e8-e1df-4408-897a-ce285a19e9d9) you might use in Excel or Google Sheets. The easiest way to understand the Filter option is as an Excel or Google Sheet formula you input within OptiSigns.

[L113] 

[L114] You can add **Rules** or **RuleSets** to your filter to create as much complexity as you need:

[L115] 

[L116] ![optisigns datasource filter with additional rules and rulesets](https://support.optisigns.com/hc/article_attachments/33468569116691)

[L117] 

[L118] In order to set up a Rule, youтАЩll need to configure three values.

[L119] 

[L120] Selecting the first box gives you these options:

[L121] 

[L122] ![optisigns datasource filter first box options](https://support.optisigns.com/hc/article_attachments/33468569122707)

[L123] 

[L124] By default, these options equate to the **headers of your columns in the spreadsheet you have mapped.** This list will vary in length depending on how many headers you have. You can also input any value you wish by typing it in the box.

[L125] 

[L126] The second box is your **Variable.** OptiSigns provides these options:

[L127] 

[L128] ![optisigns datasource filter second box options](https://support.optisigns.com/hc/article_attachments/33468569128851)

[L129] 

[L130] The final option provides the following default values:

[L131] 

[L132] ![optisigns datasource filter third box options](https://support.optisigns.com/hc/article_attachments/33468569132819)

[L133] 

[L134] By default, these map to a screen or other device, allowing your filter to target only certain screens.

[L135] 

[L136] However, this value **can be changed to anything you want.** Simply type any value you wish.

[L137] 

[L138] For our purposes, we want to check to see if the тАЬEvent Over?тАЭ is equal to No:

[L139] 

[L140] ![example of optisync datasource filter with first value set to Event Over?, second value set to Equals, third value set to No](https://support.optisigns.com/hc/article_attachments/33468569142163)

[L141] 

[L142] That removes every row where the тАЬEvent Over?тАЭ value is Yes. Now, your Event Schedule will display this:

[L143] 

[L144] ![example of daily event schedule with filters applied](https://support.optisigns.com/hc/article_attachments/33468569144851)

[L192] ### Creating a Calendar-Style Monthly Event Schedule

[L193] 

[L194] Calendar-style monthly event schedules, as the name suggests, present your event data in a calendar format. This is the best option if you want to see a great many events at a glance.

[L195] 

[L196] Should you choose the calendar-style format, space is a primary concern. YouтАЩll have to pick and choose which pieces of data you want to display - long descriptions of an event, for example, would not be able to fit on a normal sized calendar. Fortunately, you do not have to discard any of your data, nor edit your spreadsheet. All you have to do is choose what individual pieces of data you want to place on your calendar.

[L198] For this example, weтАЩll only use the name of the Lunch and the Day of the Month on the menu. You could easily add a time repeater, as well.
