[L17] 

[L18] Customers on the Enterprise Plan can now perform mass provisioning with OptiSigns. It is supported on OptiSigns Android Player, Android OS devices, Windows, Linux, Raspberry Pi, ChromeOS, or AppleTV.

[L19] 

[L20] If running a large deployment of digital signs, mass provisioning will save a lot of time and effort.

[L21] 

[L22] For example, when you use our RPI image or have MDM software to display the OptiSigns player on a device, the mass provisioning feature lets you set up the device and get it connected to your account without having to set up each device individually.

[L25] 

[L26] ## How to Set Up Mass Provisioning in the OptiSigns Portal

[L27] 

[L28] The process of performing mass provisioning is simple and straightforward.

[L29] 1. Create a provisioning template in your account.

[L30] 2. (Optional) Generate a device list. Only needed if you wish to

[L31] 3. Deploy and run.

[L32] 

[L33] ### Prerequisites:

[L34] 

[L35] * The device needs to support using a USB drive or SD card. Or there is a way you can push the configuration file to a specific location in the OS.

[L36] * The user using the OS has the privilege to run the installation and access the USB drive or SD card.

[L37] 

[L38] First time using OptiSigns? [Learn how to set up your digital signs](https://www.optisigns.com/blog/how-to-set-up-digital-signs-with-optisigns-and-amazon-fire-tv) to get a better understanding before deployment.

[L39] 

[L40] ### Step 1: Create provisioning template

[L41] 

[L42] To create a provisioning template, access it from the admin menu or use this [provisioning templates link](https://app.optisigns.com/app/s/provisioning-templates).

[L43] 

[L44] ![provisioning templates dropdown in optisigns app](https://support.optisigns.com/hc/article_attachments/34385117085715)

[L45] 

[L46] |  |

[L47] | --- |

[L48] | **NOTE:** This option will not be visible without an **Enterprise** level plan. |

[L49] 

[L50] Create a new provisioning template by clicking the **Create templates** button.

[L51] 

[L52] ![create templates button with arrow pointing at it in optisigns app](https://support.optisigns.com/hc/article_attachments/34385123847827)

[L53] 

[L54] ┬ĀIf you want to auto-step up WiFi at provisioning, you will need to create a stored WiFi first. These steps are covered later.

[L55] 

[L56] For now, set up the template in this popup:

[L57] 

[L58] ![provisioning template options in optisigns](https://support.optisigns.com/hc/article_attachments/4416555482899)

[L59] 

[L60] * Template Name: Name of your template, this is for you to distinguish it when you have multiple provisioning templates.

[L61] * Device Name Prefix: This is used to generate the device name during provisioning.

[L62] * Device Name Suffix: This is used to generate the device name during provisioning, the default setting will add timestamps as a suffix.

[L63] * Folder: The folder you want to have the provisioned devices to be created.

[L64] * WiFi: Select from the stored WiFi, which needs to be created first. Only required if you want to set up WIFI during provisioning.

[L65] * Time Zone: Specify the time zone of the device.

[L66] * Tags: Specify the tags you want to associate with the devices.

[L67] * Initial Default Content Type: Used to set the initial content on the device after provisioning.

[L68] * Orientation: Set the orientation, landscape is the default.

[L69] * Sync Play: Used to set the turn on/off of the sync play feature. For more details on the Sync Play feature, please click [here](https://support.optisigns.com/hc/en-us/articles/4412065189267-Synchronized-playback-Sync-Play-feature).

[L70] * Location: Set the location of the device.

[L71] 

[L72] Once the template is created, it will be available under the list of provisioning templates. Download the config file for use during deployment. The configuration file's name is "provisioning-template-<Your Template Name>.cfg"

[L73] 

[L74] ![provisioning templates screen with arrow pointing at file name button](https://support.optisigns.com/hc/article_attachments/4416548298259)

[L75] If you want to set up WiFi at the same time as provisioning, click the **Manage Stored WiFi** button.

[L76] 

[L77] In the popup, click "Add New WiFi", then enter the WiFi SSID and password. The WiFi will be stored and available to use in the provisioning template.

[L78] ![manage stored wifi screen with arrow pointing at add new wifi button](https://support.optisigns.com/hc/article_attachments/4416570772627)

[L79] ┬Ā

[L80] 

[L81] ### Step 2: Generate Device List (Optional)

[L82] 

[L83] If you want the device name preassigned for each device, follow this step. Otherwise, proceed to the Deployment section below.

[L84] To preassign the device name, choose **Preconfigure Devices** feature on the┬Ā**Screens**tab to create a device list with a pairing code:

[L85] ![screens tab with arrows showing how to get to preconfigure devices option](https://support.optisigns.com/hc/article_attachments/4416556453779)

[L86] 

[L87] In the popup, specify how many screens you want to preconfigure.

[L88] ![preconfigure devices options screen 1](https://support.optisigns.com/hc/article_attachments/4416556459923)

[L89] Enter a prefix┬Ā to give to your device and assign some initial content to display onscreen. The device name will be generated based on the prefix with a sequence number added. This name can be changed later before deployment.

[L90] ![preconfigure devices options screen 2](https://support.optisigns.com/hc/article_attachments/4416556506003)

[L91] Once done, the page will list all the devices preconfigured with the pairing code. Click **Done**.┬Ā This generates a CSV file named "preconfigure-devices.csv". Download it to your computer.┬Ā

[L92] ![preconfigure devices screen with two screens added](https://support.optisigns.com/hc/article_attachments/4416556695571)

[L93] Open the CSV file. It will contain the list of the preconfigured devices and their corresponding pairing code. To change the device name, simply update the CSV file. Then the file can be used in deployment if you want to have your screens using the exact pre-assigned name.

[L94] ![preconfigure devices downloaded csv file](https://support.optisigns.com/hc/article_attachments/4416571675923)

[L95] ┬Ā

[L99] ## Deployment

[L100] 

[L101] Once you have the provisioning template config file ready, you can proceed to the deployment. If you would like to pre-assign the exact name on the screen, you will need to get the preconfigured device list from step 2 as well.

[L102] 

[L103] ### OptiSigns Digital Signage Players and Windows/Linux/Android Devices

[L104] 

[L105] For mass provisioning deployment on the [OptiSigns Android Player](https://shop.optisigns.com/products/optisigns-android-stick-player-2), the [OptiSigns Pro Player](https://shop.optisigns.com/products/optisigns-digital-signage-player), or any Windows, Linux, or Android device, get a MicroSD card or USB drive and create a folder named "autoconfig" under the root directory. Place one or both of the CSV and CFG files into this folder.

[L106] 

[L107] Next, attach the MicroSD card/USB drive to your player. Launch the player, and the player will automatically detect the config file and start the provisioning process. You will receive this message once the provisioning is complete (with your local timezone configured):

[L108] 

[L109] ![mass provisioning success screen](https://support.optisigns.com/hc/article_attachments/4416680470163)

[L110] 

[L111] ### Raspberry Pi Devices

[L112] 

[L113] Raspberry Pi devices have two options for mass deployment: the USB method detailed above, or via SSH.

[L114] 

[L115] For more on the SSH method, see [how to enable SSH for your Raspberry Pi](https://support.optisigns.com/hc/en-us/articles/1500008008981-How-to-enable-SSH-for-your-Raspberry-Pi) and [how to mass deploy OptiSigns on Raspberry Pi devices](https://support.optisigns.com/hc/en-us/articles/360063098753-How-to-mass-deploy-OptiSigns-on-Raspberry-Pi-devices).

[L116] 

[L117] ### Apple TV and ChromeOS

[L118] 

[L119] Deployment on an Apple TV or a ChromeOS device has to be done through their respective platforms.
