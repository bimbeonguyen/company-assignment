[L1] # GraphQL API Tutorial: Pair and Assign Content to Screen

[L2] 

[L3] In this tutorial, we will walk through how to get a new screen paired and assign content making it ready to use through the API. Generally, it involves 2 steps. ┬ĀFirstly, you will need to get the screen paired to your account. Then you will need to update the screen to rename it, and assign the content to be played on the screen.

[L4] 

[L5] #### **1. Pair screen**

[L6] 

[L7] To pair a screen, you will need to use the pairDevice mutation. And supply the pairing code you get from the screen as a arguments in the payload. If it runs successfully, it will return the requested data from the server, and an id will be assigned to the screen. This id will be needed in the next step to rename and assign the content to the screen.

[L8] 

[L9] ```

[L10] mutation {  

[L11]   pairDevice(payload:{pairingCode:"3JRKC8"}){  

[L12]           _id,  

[L13]           deviceName,  

[L14]           UUID,  

[L15]           pairingCode,  

[L16]           currentType,  

[L17]           currentAssetId,  

[L18]           localAppVersion  

[L19]   }  

[L20] }

[L21] ```

[L25] #### **2. Rename the paired screen and assign asset/playlist to the screen**

[L26] 

[L27] To change the screen name or change the content assigned to the screen, you can use the updateDevice mutation. You will need to supply the screen id retrieved from the above step, and specify how you would like to change the device in the payload.

[L28] 

[L29] ```

[L30] mutation {  

[L31]   updateDevice(_id:"6682d6d553fca60012953e17",payload: {deviceName: "GraphAPI Test",  

[L32]     currentType: ASSET,  

[L33]     currentAssetId: "uRQynMhDsJ6QY35Wf",  

[L34]     orientation: LANDSCAPE}){  

[L35]           _id,  

[L36]           deviceName,  

[L37]           UUID,  

[L38]           pairingCode,  

[L39]           currentType,  

[L40]           currentAssetId,  

[L41]           localAppVersion  

[L42]   }  

[L43] }

[L44] ```

[L45] 

[L46] ![](https://support.optisigns.com/hc/article_attachments/36564341502611)

[L47] 

[L48] To assign a playlist to the screen, it is about the same as assign asset to screen. Just need to change the currentType to PLAYLIST, and supply the playlist id.

[L49] 

[L50] ```

[L51] mutation {  

[L52]   updateDevice(_id:"6126edf99834540019b30ff1",payload: {deviceName: "GraphAPI Test",  

[L53]     currentType: PLAYLIST  

[L54]     currentAssetId: "d87B9ARKPyH8YYBbs",  

[L55]     orientation: LANDSCAPE}){  

[L56]           _id,  

[L57]           deviceName,  

[L58]           UUID,  

[L59]           pairingCode,  

[L60]           currentType,  

[L61]           currentAssetId,  

[L62]           localAppVersion  

[L63]   }  

[L64] }

[L65] ```
