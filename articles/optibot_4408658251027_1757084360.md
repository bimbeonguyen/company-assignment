[L2] 

[L3] ### In this article, we'll show you how to use the OptiSigns Portal to Execute Remote Commands on your Windows and Linux Devices.

[L5] * [Enable Remote Command Feature on Account](#Enable)

[L11] | **NOTE** |

[L12] | In order to use this feature, you will need a **Pro Plus** **or above plan**. |

[L13] 

[L14] The Execute Remote Commands feature allows you to remotely manage screens running on Windows/Linux/Raspberry Pi. It allows command to be pushed remotely to individual devices or groups of devices simultaneously.

[L15] 

[L16] ---

[L17] 

[L18] ## Enable Remote Command Feature on Account

[L19] 

[L20] Remote Command Execution is disabled by default. To use it, it will need to be turned on. To do this, navigated to [the Preferences section](https://app.optisigns.com/app/s/preference-settings) in the OptiSigns portal.

[L21] 

[L22] |  |

[L23] | --- |

[L24] | For security reasons, **only the account owner** can enable or disable Remote Command Execution. |

[L30] ## How to Execute Remote Commands

[L31] 

[L32] To submit the command to manage the devices remotely, you will need to go to the remote command execution console. Click **Screens**, then the **More Options (Three Dots)**

[L33] 

[L34] Screens -> More Options Icon -> Execute Remote Commands

[L35] 

[L36] ![how to get to execute remote commands console](https://support.optisigns.com/hc/article_attachments/36078232655763)

[L37] 

[L38] The Execute Remote Commands console can be broken down into three sections:

[L39] 

[L40] * **Section 1** - Where you specify the target of the commands.

[L41] * **Section 2** - The command you want to execute.

[L42] * **Section 3** - The execution result and history.

[L43] 

[L44] ![three sections of execute remote commands](https://support.optisigns.com/hc/article_attachments/36078232658323)

[L45] 

[L46] For section 1, there are 2 types of targets you can choose from:

[L47] 

[L48] * **Screens** - You can select the screen name here, it can be one screen or multiple screens.

[L49] * **Tags** - Utilizing tags, you can execute the command on a group of devices. In the below example, the command will be submitted to all devices tagged as Windows or Raspberry Pi.

[L50] 

[L51] ![remote commands tags](https://support.optisigns.com/hc/article_attachments/4408658532371)

[L52] 

[L53] For section 2, you can enter the command you want to execute in the text box. The command needs to be OS-specific, depending on the OS your device is running on, you will need to build the scripts accordingly. Once you have your commands ready, just click the submit button. The command will be pushed to the devices for execution.

[L54] 

[L55] Other than the free-form scripts, there are some common commands built into the platform, to achieve these functions, you will just need to select it from the drop-down menu. Depending on the type of device OS, the command will be submitted accordingly.

[L56] 

[L57] ![common remote commands](https://support.optisigns.com/hc/article_attachments/36078208103315)

[L58] 

[L59] After a command is executed, it will appear in section 3: the command history.

[L67] 

[L68] ## Controlling Remote Commands Received by the Device

[L69] 

[L70] In some cases, you may not want some of your devices to be controlled remotely for any reason. The ability to receive remote commands can be toggled on the device.

[L71] 

[L72] Go to the side menu on the OptiSigns player, click **Advanced Options**, and you'll find the toggle for **Execute Remote Commands**. Toggling this off will cause the device to reject remote commands.

[L73] 

[L74] ![device enable remote commands toggle](https://support.optisigns.com/hc/article_attachments/36078208107539)
