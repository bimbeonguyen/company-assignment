[L1] # Tutorial: Creating or Updating Website Assets Using GraphQL

[L2] 

[L3] Using GraphQL, it is possible to create a website asset on OptiSigns.

[L4] 

[L5] |  |

[L6] | --- |

[L7] | **NOTE** |

[L8] | At this time, this Mutation is primarily used to create or update Website assets. |

[L9] 

[L10] **1. Creating a New Website Asset**

[L11] 

[L12] To do this, weŌĆÖll need to create a specific Mutation:

[L13] 

[L14] ```

[L15] mutation saveAsset ($payload: AssetInput!, $teamId: String)  

[L16] {saveAsset (payload:$payload,teamId:$teamId){  

[L17]   _id  

[L18]   originalFileName  

[L19]   webLink  

[L20]   webType  

[L21]   fileType  

[L22]   }  

[L23] }

[L24] ```

[L25] 

[L26] **Variables:**

[L27] 

[L28] ```

[L29] {  

[L30]   "payload": {  

[L31]     "originalFileName": "",  

[L32]     "webLink": "",  

[L33]     "webType": "",  

[L34]     "fileType": ""  

[L35]   }  

[L36] }

[L37] ```

[L38] 

[L39] ![](https://support.optisigns.com/hc/article_attachments/36562094972435)

[L40] 

[L41] This Mutation, once run, will create a new asset:

[L42] 

[L43] ![](https://support.optisigns.com/hc/article_attachments/36562091083539)

[L44] 

[L45] This asset can now be assigned to a device, queried for additional information, or added to a playlist or schedule.

[L46] 

[L47] **2. Updating Assets**

[L48] 

[L49] Updating assets is as simple as running the same Mutation while providing an \_id field. This can be done for any asset type, not just website assets, but we will be using a website asset as an example.

[L50] 

[L51] **Variables:**

[L52] 

[L53] ```

[L54] {  

[L55]   "payload": {  

[L56]  ┬Ā  "_id": "",  

[L57]     "originalFileName": "",  

[L58]     "webLink": "",  

[L59]     "webType": "",  

[L60]     "fileType": ""  

[L61]   }  

[L62] }

[L63] ```
