[L1] # How to set up SAML 2.0 with OptiSigns and Google Workspace

[L2] 

[L3] With Pro Plus and Enterprise plan, you can configure SAML 2.0 with OptiSigns via Google Workspace. The Google Workspace will be acting as the IDP (Identify Provider), and OptiSigns will be working as the SP(Service Provider).

[L4] 

[L5] ### **Set up OptiSigns & Google Workspace:**

[L6] 

[L7] **First you need to do some set up in OptiSigns:**

[L8] 

[L9] If you don't have a sub domain yet, you can set up one by going to:  

[L10] <https://app.optisigns.com/app/s/branding-settings>

[L11] 

[L12] Fill in subdomain field and click Activate. After that you can use this sub domain for "  

[L13] You can also map your own domain like digitalsigns.yourcompany.com by following this┬Ā[article](https://support.optisigns.com/hc/en-us/articles/1500000480302).

[L14] 

[L15] This will be the URL that you can share with your users so they can log in to use the app, once integration has set up. In our example we will use┬Ā<https://advanced.optisigns.net/>

[L16] 

[L17] ![mceclip13.png](https://support.optisigns.com/hc/article_attachments/21287201525907)

[L18] 

[L19] Next, go to the SAML Single Sign On setting page:

[L20] 

[L21] <https://app.optisigns.com/app/s/saml-settings>

[L22] 

[L23] Click Enable SAML SSO.

[L24] 

[L25] The settings are:

[L26] 

[L27] * Enable Username & Password login: Allow users to also log in with username/password. ItŌĆÖs recommended to disable it once the integration is all done. As Admin/Owner, it's recommended that you keep at least 1 account with a password log in, in case there are issues, you can always log back in from app.optisigns.com to reconfigure.

[L28] * Enable User Creation:┬ĀIf users are authenticated, but do not exist in OptiSigns, they will be created in OptiSigns. You should enable this, because you likely already assign/approve users/groups to use OptiSigns, unless for some reason you want to be very strict and want to review roles of users before they can start using OptiSigns.

[L29] * Enable User Override:┬ĀEvery time a user logs in, if their group assignment have changed on SAML, OptiSigns will update, override new profile settings.

[L30] * Note the "Single Sign On URL" and "Audience URI (SP Entity ID) URL", you will need this to use in Google Workspace later.

[L31] 

[L32] ![](https://support.optisigns.com/hc/article_attachments/21286640961939)

[L33] 

[L34] ┬Ā

[L35] 

[L36] **Next, add OptiSigns as an App in your Google Workspace admin portal:**

[L37] 

[L38] Log in to your Google Workspace portal as admin -> Apps -> Web and mobile apps

[L39] 

[L40] Click Add app -> Add custom SAML app ![mceclip0.png](https://support.optisigns.com/hc/article_attachments/4407485683475)

[L41] 

[L42] In the popup window, enter OptiSigns as the name of the app, you can upload the app icon here as well. Then click continue.

[L43] 

[L44] ![mceclip1.png](https://support.optisigns.com/hc/article_attachments/4407485688467)

[L45] 

[L46] The next page will provide the IDP data. Get these 2 highlighted information then click continue, these need to be maintained in the OptiSigns SAML SSO settings later.

[L47] 

[L48] ![](https://support.optisigns.com/hc/article_attachments/21287044557843)

[L49] 

[L50] Next page will be the SP information, this is where you should provide the Single Sign On URL, and SP Entity ID you get from your OptiSigns SAML SSO setting.

[L51] 

[L52] SP Entity ID from OptiSigns SAML SSO setting should be put under Entity ID.

[L53] 

[L54] Single Sign On URL from OptiSigns SAML SSO setting should be put under ACS URL.

[L55] 

[L56] Also, set the Name ID format to Email.

[L57] 

[L58] ![](https://support.optisigns.com/hc/article_attachments/21286792807699)

[L59] 

[L60] The next page is where you maintain the attributes. This step will be explained later in this article. Click Finish, and the app is added to the Google Workspace.

[L61] 

[L62] ![](https://support.optisigns.com/hc/article_attachments/21286840258963)

[L63] 

[L64] After completing the app creation on Google Workspace. You can select the OptiSigns app under the "Web and mobile apps". Click on the OptiSigns app, and note the ID in the URL, that is the SPID that will be needed.

[L65] 

[L66] ![mceclip7.png](https://support.optisigns.com/hc/article_attachments/4407493347731)

[L67] 

[L68] Go back to your OptiSigns account maintain above mentioned 3 fields, and save it.

[L69] 

[L70] You can get SSO URL, Entity ID, and Certificate from your Google Workspace.

[L71] 

[L72] * Put SSO URL from Google Workspace under SAML 2.0 Endpoint (HTTP).

[L73] * Put Entity ID from Google Workspace under Identity Provider Issuer.

[L74] * Put the content from the base64 encoded certificate under Public Certificate.

[L75] 

[L76] ![](https://support.optisigns.com/hc/article_attachments/21287166076819)

[L77] 

[L78] Now your login portal & integration are all setup.

[L82] #### **Assign & map users, and groups from Google Workspace to OptiSigns**

[L83] 

[L84] It's not required, but recommended to create groups of users to be assigned, and map to OptiSigns Roles, and Teams so they will automatically have the right role & group.

[L85] 

[L86] **IMPORTANT NOTE: If you don't configure this, all users will be assigned User Role & Default Team (screenshot see below)**

[L87] 

[L88] ┬Ā

[L89] 

[L90] To configure how OptiSigns should map the user groups to OptiSigns Roles by going to:┬Ā<https://app.optisigns.com/app/s/saml-settings>

[L91] 

[L92] Scroll to Advanced Settings and create a mapping.  

[L93] Group Name (can use department from Google Workspace), Role (role in OptiSigns) mapping.┬Ā  

[L94] ![mceclip8.png](https://support.optisigns.com/hc/article_attachments/4407485832851)

[L95] 

[L96] It's best practice to create a group specifically for OptiSigns with name prefix with optisigns- and map to OptiSigns like below:

[L97] 

[L98] * optisigns-admins (SAML group) -> OptiSigns role: Admin

[L99] * optisigns-users (SAML group) -> OptiSigns role: Users

[L100] * optisigns-custom-role (SAML group) -> OptiSigns custom role that you create

[L101] 

[L102] **How to handle Unmapped users/groups:**

[L103] 

[L104] You can map the "Unmapped users/group" to No Team (Disabled)

[L105] 

[L106] This way they will receive an error when trying to log in and will have to reach out to Admins to get correct teams, and roles assigned. This can be used as a safe guard, in case some users accidentally got assigned OptiSigns app but not the right groups.

[L107] 

[L108] ![mceclip3.png](https://support.optisigns.com/hc/article_attachments/21287201539475)

[L109] 

[L110] Note that if you map a SAML group to a Team and then delete the team, it will result in new user being mapped to No Team and will have to contact you to be assigned to a team to use the app.

[L111] 

[L112] ┬Ā

[L113] 

[L114] Next, it is time to talk about the attributes mapping. This is the last step when creating the app in Google Workspace.

[L115] 

[L116] Currently, OptiSigns support attributes mapping of first name, last name and group. You can define the attribute name in Google Workspace and set it to the same default attributes name used on OptiSigns.

[L117] 

[L118] These mappings will pass information to OptiSigns on what's user's Name and Groups.

[L119] 

[L120] ![mceclip5.png](https://support.optisigns.com/hc/article_attachments/4407485726611)

[L121] 

[L122] ┬Ā

[L123] 

[L124] The "App attributes" are corresponding to OptiSigns

[L125] 

[L126] <https://app.optisigns.com/app/s/saml-settings>

[L127] 

[L128] OptiSigns accept firstName, lastName, and group by default. Instead of setting the attribute names to the default attribute name used on OptiSigns,┬Ā you can also change the attribute name on OptiSigns to match the attribute name you defined on Google Workspace as well.

[L129] 

[L130] ![mceclip9.png](https://support.optisigns.com/hc/article_attachments/4407485875347)
