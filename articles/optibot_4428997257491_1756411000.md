[L1] # How to organize templates and share across teams 

[L2] 

[L3] When there is a large number of templates and a complex structure of teams/departments, how to manage templates and share them with others could be difficult. For example, there are hundreds of templates of different themes, some should only be available to team A and some should be available to other teams. How that should be managed with OptiSigns?

[L4] 

[L5] The recommended approach is to leverage the folder structure and folder permissions to organize the templates and manage the authorization.┬Ā

[L6] 

[L7] In general, it is mainly 3 parts.

[L8] 

[L9] * Set up the proper folder/sub-folder structure to organize the templates, so it is easier to navigate.

[L10] * Configure the permissions on the folder to have it assigned to the right teams. Since the template cannot be modified, the folder should be set up with "View" permission only.

[L11] * When others use the template, they will not have permission to save and change the template, instead, they will have to "save as" at another location where they have permission.

[L12] 

[L13] Here is an example. The "QR\_Code\_Template" is under TemplateFolder / TemplateSubFolder. To make it only viewable but not modifiable by others, you will need to set the folder permission to view only.┬Ā In this case, the folder is available to Team 1. So users from Team1 can open and use the template, but they cannot modify the template and have to save it to another location where they have permission.┬Ā┬Ā

[L14] 

[L15] ![mceclip0.png](https://support.optisigns.com/hc/article_attachments/4428983032467)

[L16] 

[L17] Determine who should have permission can be managed through the folder permission setting. Since this is used as a template, the permission should be viewed only by others.

[L18] 

[L19] ![mceclip1.png](https://support.optisigns.com/hc/article_attachments/4428989990419)

[L20] 

[L21] The template will be available to users from Team1.┬Ā

[L22] 

[L23] ![mceclip2.png](https://support.optisigns.com/hc/article_attachments/4428994050835)

[L24] 

[L25] The users will be able to open the template and make changes, however when they try to save it, it will be declined because of permission. Users have to use click "Save As" to save it to a location they have permission.┬Ā

[L26] 

[L27] ![mceclip3.png](https://support.optisigns.com/hc/article_attachments/4429003896083)

[L28] 

[L29] Users can save it to a location they have edit permission by choosing a folder and clicking the "Save As" button.
