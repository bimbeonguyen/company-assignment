[L1] # How to set up SAML 2.0  with OptiSigns and Okta

[L2] 

[L3] With Pro Plus and Enterprise plans, you can configure SAML 2.0 with OptiSigns via Okta.

[L4] 

[L5] Assuming you already using Okta for identity management. If you have not used Okta, it is the leading identity management platform, you can learn more [here](https://www.okta.com/intro-to-okta/).

[L6] 

[L7] ┬Ā

[L8] 

[L9] ### **Set up OptiSigns & Okta:**

[L10] 

[L11] **First, you need to do some setup in OptiSigns:**

[L12] 

[L13] If you don't have a subdomain yet, you can set up one by going to:  

[L14] <https://app.optisigns.com/app/s/branding-settings>

[L15] 

[L16] Fill in the subdomain field and click Activate. After that, you can use this subdomain for "  

[L17] You can also map your domain like digitalsigns.yourcompany.com by following this [article](https://support.optisigns.com/hc/en-us/articles/1500000480302).

[L18] 

[L19] This will be the URL that you can share with your users so they can log in to use the app, once integration has set up. In our example, we will use <https://advanced.optisigns.net/>

[L20] 

[L21] ![](https://support.optisigns.com/hc/article_attachments/43875570896659)

[L22] 

[L23] Next, go to the SAML Single Sign On setting page:

[L24] 

[L25] <https://app.optisigns.com/app/s/saml-settings>

[L26] 

[L27] Click Enable SAML SSO.

[L28] 

[L29] The settings are:

[L30] 

[L31] * Enable Username & Password login: Allow users to also log in with username/password. ItŌĆÖs recommended to disable it once integration is all done. As Admin/Owner, it's recommended that you keep at least 1 account with a password log in, in case there's issues, you can always log back in from app.optisigns.com to reconfigure.

[L32] * Enable User Creation: If users are authenticated, but do not exist in OptiSigns, they will be created in OptiSigns. You should enable this, because you likely already assign/approve users/groups to use OptiSigns, unless for some reason you want to be very strict and want to review the roles of users before they can start using OptiSigns.

[L33] * Enable User Override: Every time a user logs in, if their group assignment has changed on SAML, OptiSigns will update, and override new profile settings.

[L34] * Note the "Single Sign On URL" and "Audience URI (SP Entity ID) URL", you will need this to use in Okta later.

[L35] 

[L36] ![](https://support.optisigns.com/hc/article_attachments/28207766414227)

[L39] 

[L40] **Next, add OptiSigns as an App to your Okta account:**

[L41] 

[L42] Log in to your Okta account as admin -> Application

[L43] 

[L44] Or go to: <https://optisigns-admin.okta.com/admin/apps/active>

[L45] 

[L46] Click Create App Integration

[L48] ![mceclip0.png](https://support.optisigns.com/hc/article_attachments/4404598581779)

[L50] Select SAML 2.0

[L51] 

[L52] ![mceclip1.png](https://support.optisigns.com/hc/article_attachments/4404590829203)

[L53] 

[L54] Enter App name: OptiSigns

[L55] 

[L56] If you want to upload a logo, you can use our logo [here](https://download.optisignsapp.com/marketing/optisigns-logo.png).

[L57] 

[L58] Click Next

[L59] 

[L60] ![mceclip2.png](https://support.optisigns.com/hc/article_attachments/4404590844691)

[L61] 

[L62] In "Single sign-in URL" and "Audience URI (SP Entity ID)", these are the URL that you have in <https://app.optisigns.com/app/s/saml-settings>  

[L63] Check "Use this for Recipient URL and Destination URLs"

[L64] 

[L65] Click Next.

[L66] 

[L67] ![](https://support.optisigns.com/hc/article_attachments/52406311133459)![](https://support.optisigns.com/hc/article_attachments/28207766417811)

[L68] 

[L69] The last step is just informational for Okta to know how you are using the app.

[L70] 

[L71] Select "I'm an Okta customer adding an internal app" as OptiSigns is now an internal app in your organization.

[L72] 

[L73] Click Next.

[L74] 

[L75] ![mceclip8.png](https://support.optisigns.com/hc/article_attachments/4404615741075)

[L76] 

[L77] ┬Ā

[L78] 

[L79] Then click "View Setup Instruction"

[L80] 

[L81] Copy these 3 fields and paste into OptiSigns' SAML config page:

[L82] 

[L83] * Identity Provider Single Sign-On ULR -> OptiSigns: SAML 2.0 Endpoint (HTTP)

[L84] * Identity Provider Issuer -> OptiSigns: Identity Provider Issuer

[L85] * X.509 Certificate -> OptiSigns: Public Certificate

[L86] 

[L87] ![okta-config.png](https://support.optisigns.com/hc/article_attachments/4404615786259)

[L88] 

[L89] ┬Ā

[L90] 

[L91] Lastly, set the "Sign In Button label", this is the text of the button you want your users to see in their login portal. Use something descriptive like "Log in with Okta" "Sign in with SSO" or something your user is familiar with.

[L92] 

[L93] ![okta-config-2.png](https://support.optisigns.com/hc/article_attachments/4404615885331)

[L94] 

[L95] Click Save

[L96] 

[L97] Now your login portal & integration are all setup.

[L98] 

[L99] Next, you need to assign users, and groups that can use OptiSigns.

[L100] 

[L101] ┬Ā

[L102] 

[L103] #### **Assign & map users, and groups from Okta to OptiSigns**

[L104] 

[L105] It's not required, but recommended to create groups of users to be assigned, and map to OptiSigns Roles, and Teams so they will automatically have the right role & group.

[L106] 

[L107] **IMPORTANT NOTE: If you don't configure this, all users will be assigned User Role & Default Team (screenshot see below)**

[L108] 

[L109] ┬Ā

[L110] 

[L111] To configure how OptiSigns should map the user groups to OptiSigns Roles by going to: <https://app.optisigns.com/app/s/saml-settings>

[L112] 

[L113] Scroll to Advanced Settings and create a mapping.  

[L114] Group Name (group names in Okta), Role (role in OptiSigns) mapping.┬Ā  

[L115] ![mceclip1.png](https://support.optisigns.com/hc/article_attachments/4404820186003)

[L116] 

[L117] It's best practice to create a group name prefix with Optisigns- and map to OptiSigns like below:

[L118] 

[L119] * optisigns-admins (SAML group) -> OptiSigns role: Admin

[L120] * optisigns-users (SAML group) -> OptiSigns role: Users

[L121] * optisigns-custom-role (SAML group) -> OptiSigns custom role that you create

[L122] 

[L123] **How to handle Unmapped users/groups:**

[L124] 

[L125] You can map the "Unmapped users/group" to No Team (Disabled)

[L126] 

[L127] This way they will receive an error when trying to log in and will have to reach out to Admins to get the correct teams, and roles assigned. This can be used as a safeguard, in case some users accidentally got assigned OptiSigns app but not the right groups.

[L128] 

[L129] ![mceclip3.png](https://support.optisigns.com/hc/article_attachments/4404812977171)

[L130] 

[L131] Note that if you map a SAML group to a Team and then delete the team, it will result in new user being mapped to No Team and will have to contact you to be assigned to a team to use the app.

[L139] Click Assign -> People or Groups to use this app. You can also configure your user to request to use the app, but that's beyond the scope of this article.

[L143] You can set up group mapping by going to General -> SAML settings

[L145] ![mceclip2.png](https://support.optisigns.com/hc/article_attachments/4404762241811)

[L155] The "Attribute Statement" and "Group Attribute Statement" are corresponding to OptiSigns <https://app.optisigns.com/app/s/saml-settings>

[L156] 

[L157] OptiSigns accept firstName, lastName, and group by default, but if you change these in Okta, you will need to match it here as well.
