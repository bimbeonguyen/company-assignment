[L1] # Handle OAuth Authentication using API Gateway Pre-request Configuration

[L2] 

[L3] ### In this article, we will explain how to set up a Pre-request to retrieve an OAuth 2.0 access token for connecting to an API using an API Gateway.

[L4] 

[L5] OptiSigns API Gateway allows for OAuth authentication via Pre-request. This gives users the capability to consume API that requires OAuth authentication or similar.

[L6] 

[L7] To get started, you'll need to set up an API request. Hover over **Account Name ŌåÆ More****ŌåÆ** Click **DataSources**:

[L8] 

[L9] ![how to navigate to datasource](https://support.optisigns.com/hc/article_attachments/39080869728915)

[L10] 

[L11] From there, hit **Add Request**.

[L12] 

[L13] ![datasources add request button](https://support.optisigns.com/hc/article_attachments/39080900410515)

[L14] 

[L15] Create a **GET****Request** and input your API endpoint, then click **Pre-request:**

[L16] 

[L17] ![optisigns api request form pre-request](https://support.optisigns.com/hc/article_attachments/39080869729555)

[L18] 

[L19] Within the Pre-request field, input the following code:

[L20] 

[L21] ```

[L22] const body = {  

[L23]   "grant_type": "client_credentials",  

[L24]   "client_id": "<CLIENT_ID>",  

[L25]   "client_secret": "<CLIENT_SECRET>"  

[L26]   

[L27] };  

[L28] const params = Object.keys(body || {}).map((key) => {  

[L29]         return key + '=' + body[key];  

[L30]       }).join('&');  

[L31]   

[L32] const {data, headers} = await os.postRequest("<OAUTH_AUTHENTICATION_URL>", params,{headers: {'content-type': 'application/x-www-form-urlencoded'}});  

[L33] const token = 'Bearer' + data.access_token;  

[L34] os.context.set("request.headers.authorization", token);

[L35] ```

[L36] 

[L37] **Notes:**

[L38] 

[L39] * "grant\_type": Use "client\_credentials" ., because "client\_credentials" is the grant type in OAuth for server-side integration without user interaction.

[L40] * <CLIENT\_ID> and <CLIENT\_SECRET> refers to the user's code for the API being accessed, this will need to be provided by the user.

[L41] * <OAUTH\_AUTHENTICATION\_URL> refers to the URL the access token is being retrieved from. This URL will need to be provided by the user.

[L42] 

[L43] Now configure the **Header**:

[L44] 

[L45] ![properly configured API header](https://support.optisigns.com/hc/article_attachments/39080900411795)

[L46] 

[L47] With this and the rest of the required fields filled out, you've properly configured your Pre-request. Hitting┬Ā**Run Test** should return a **200 OK** Response.

[L48] 

[L49] ![successful api request](https://support.optisigns.com/hc/article_attachments/39080869736211)

[L50] 

[L51] If so, hit┬Ā**Save**to finish your API Request.
