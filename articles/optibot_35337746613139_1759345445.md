[L15] It is common for companies to share their internal communications, including memos, news, announcements and more via intranet. Digital signage will heighten employee awareness of published company news and announcements.

[L16] 

[L17] Using OptiSync, itŌĆÖs possible to create a custom news app to display company news stored on your companyŌĆÖs intranet. This is achieved by connecting an intranet API with OptiSync and choosing what data to display in the [OptiSigns Designer app](https://support.optisigns.com/hc/en-us/articles/4404151402899-How-to-use-OptiSigns-Template-Designer-app-to-make-your-Digital-Signs-in-minutes).

[L18] 

[L19] Creating news feeds with OptiSync has a key advantage: you have full control of the look and feel of your feed. You can create feeds that match your company branding and design guidelines no matter which news source you use. We will be using SharePoint here as an example, but you can use anything that can be input as a Datasource.

[L67] ### Step 1: Setting Up a Custom API Gateway (SharePoint Example)

[L68] 

[L69] We cover most of the details of this step in our [How to Integrate API and Publish API Data via OptiSync](https://support.optisigns.com/hc/en-us/articles/22875592994195-How-to-Integrate-API-and-Publish-API-Data-via-OptiSync) guide. Please see that guide for a step-by-step process.

[L70] 

[L71] However, there are a few elements which are specific to connecting to a SharePoint API (or a variety of resources in Microsoft 365), which we will cover here.

[L72] 

[L73] Setting up this API gateway will consist of two parts:

[L74] 

[L75] * Configuring a Pre-request call to obtain an Authentication token

[L76] * Using that Authentication token in a "main call" to obtain your SharePoint data.

[L77] 

[L78] #### Obtaining a Site-Specific SharePoint SiteID Value

[L79] 

[L80] The first step in this process is obtaining the SiteID value for your SharePoint site. This is necessary to obtain the proper URL to make the API request.

[L81] 

[L82] To do this, download┬Ā[**Postman**](https://www.postman.com/)or another API platform. Then, send a GET request to the following URL:

[L83] 

[L84] ```

[L85] https://graph.microsoft.com/v1.0/sites/[YOUR-SHAREPOINT-SITE-URL]

[L86] ```

[L87] The [YOUR-SHAREPOINT-SITE-URL] will be specific to your SharePoint site.

[L88] 

[L89] When configured correctly, this API request should return a string of JSON similar to the below:

[L90] 

[L91] ![](https://support.optisigns.com/hc/article_attachments/45228746837523)

[L92] 

[L93] As you can see, under the "id" section there are two values. You want the string after the URL. That is your┬Ā**SiteID**value. This will be critical for setting up your API gateway in the next step.

[L94] 

[L95] #### Retrieving Proper Authentication Tokens from Microsoft

[L96] 

[L97] Authentication for SharePoint can be carried out using different methods, but OAuth 2.0 has become the recommended standard. To get started using OAuth 2.0, youŌĆÖll need to [register your application in Azure Active Directory](https://learn.microsoft.com/en-us/azure/healthcare-apis/register-application).

[L98] 

[L99] Once that's done, you'll need:

[L100] 

[L101] * [**Client ID**](https://learn.microsoft.com/en-us/azure/healthcare-apis/register-application#application-id-client-id)

[L102] * [**Client Secret Value**](https://learn.microsoft.com/en-us/azure/healthcare-apis/register-application#certificates--secrets)

[L103] * The **SiteID** from the previous step

[L104] 

[L105] These are required for authentication. Any further authentication can be done in the Pre-Request stage to obtain access tokens. These authentication tokens will need to be kept refreshed: see [Microsoft's article on identity platform certificate credentials](https://learn.microsoft.com/en-us/entra/identity-platform/certificate-credentials) for more information.

[L106] 

[L107] With these pieces of information, we can get started. Create a new DataSource Request (see our article on [How to Integrate API and Publish API Data via OptiSync](https://support.optisigns.com/hc/en-us/articles/22875592994195-How-to-Integrate-API-and-Publish-API-Data-via-OptiSync) on how to do this). Set up a **GET** request with the following URL.┬Ā

[L108] 

[L109] ```

[L110] https://graph.microsoft.com/v1.0/sites/[YOUR-SITEID-VALUE]/pages/microsoft.graph.sitePage?$filter=promotionKind eq 'newsPost'

[L111] ```

[L112] Input your SiteID value where relevant. Next, we'll set up the headers.

[L113] 

[L114] ##### **Headers Configuration**

[L115] 

[L116] For the request headers, use these:

[L117] 

[L118] ![](https://support.optisigns.com/hc/article_attachments/45228732984467)

[L119] 

[L120] The "Bearer" field can be left blank, as this value will be obtained in the Pre-Request stage.

[L121] 

[L122] ##### **Pre-Request Configuration**

[L123] 

[L124] The Pre-Request stage is where we will obtain the Authorization token for the request. It will need to be set up in a very specific manner. Input the below code in the Pre-Request field:

[L125] 

[L126] ```

[L127] const body = {

[L128]   "grant_type": "client_credentials",

[L129]   "client_id": "[YOUR-CLIENT-ID]",

[L130]   "client_secret": [YOUR-SECRET-VALUE],

[L131]   "scope":"https://graph.microsoft.com/.default"

[L132] };

[L133] 

[L134] const urlEncodedBody = Object.keys(body || {}).map((key) => {

[L135]                 return key + '=' + body[key];

[L136]             }).join('&');

[L137] 

[L138] const {data, headers} = await os.postRequest("https://login.microsoftonline.com/[YOUR-CLIENT-ID]/oauth2/v2.0/token", urlEncodedBody,{headers: {'content-type': 'application/x-www-form-urlencoded'}});

[L139] 

[L140] const token = 'Bearer ' + data.access_token;

[L141] os.context.set("request.headers.authorization", token);

[L142] 

[L143] ```

[L144] Replace the areas marked [YOUR-CLIENT-ID] and [YOUR-SECRET-VALUE] with your Client ID and Secret Value, respectively.

[L145] 

[L146] ##### **Final Request and Testing**

[L147] 

[L148] With these configured, your request should look like this:

[L149] 

[L150] ![](https://support.optisigns.com/hc/article_attachments/45229015845523)You can hit the **Run Test** button at the bottom to test the connection. If successful, it should return a **200 OK** response and provide JSON data.

[L151] 

[L152] ![](https://support.optisigns.com/hc/article_attachments/45228746849171)

[L153] 

[L154] Be sure to **Enable this Request**, then **Save** your API Request before moving on to the next step.

[L155] 

[L156] ![](https://support.optisigns.com/hc/article_attachments/45228746851347)

[L252] ### Applying Filters to Customize Which News Stories to Display

[L253] 

[L254] What news stories you wish to display may vary depending on a number of factors. In order to filter out undesired or redundant stories, you can make use of the OptiSigns **DataSource** **Filter**.

[L255] 

[L256] To get started, highlight the data you want to filter, then hit **Settings.** Next, hit the **Filter** option under your DataSource.

[L257] 

[L258] ![firefox_yyCyvxGXBi.jpg](https://support.optisigns.com/hc/article_attachments/43089441904659)

[L259] 

[L260] This screen will appear:

[L261] 

[L262] ![firefox_M6EvhPuhbI.jpg](https://support.optisigns.com/hc/article_attachments/43089441904915)

[L263] 

[L264] What follows is, essentially, an [AND statement](https://support.microsoft.com/en-us/office/and-function-5f19b2e8-e1df-4408-897a-ce285a19e9d9) or an [OR statement](https://support.microsoft.com/en-us/office/and-function-5f19b2e8-e1df-4408-897a-ce285a19e9d9) you might use in Excel or Google Sheets. The easiest way to understand the Filter option is as an Excel or Google Sheet formula you input within OptiSigns.

[L265] 

[L266] You can add **Rules** or **RuleSets** to your filter to create as much complexity as you need:

[L267] 

[L268] ![datasource filter option rulesets](https://support.optisigns.com/hc/article_attachments/35337756575507)

[L269] 

[L270] In order to set up a Rule, youŌĆÖll need to configure three values.

[L271] 

[L272] Selecting the first box gives you these options:

[L273] 

[L274] ![filter first box options](https://support.optisigns.com/hc/article_attachments/35337756579731)

[L275] 

[L276] By default, these options equate to the **headers of the data mapped to the source.** This list will vary in length depending on how many headers you have. You can also input any value you wish by typing it in the box.

[L277] 

[L278] The second box is your **Variable.** OptiSigns provides these options:

[L279] 

[L280] ![filter variable data](https://support.optisigns.com/hc/article_attachments/35337746579347)

[L281] 

[L282] The final option provides the following default values:

[L283] 

[L284] ![filter final options](https://support.optisigns.com/hc/article_attachments/35337756591251)

[L285] 

[L286] By default, these map to a screen or other device, allowing your filter to target only certain screens.

[L287] 

[L288] However, this value **can be changed to anything you want.** Simply type any value you wish.

[L289] 

[L290] There are dozens of possibilities using the OptiSync filter to show even more precise automated data on your screens.
