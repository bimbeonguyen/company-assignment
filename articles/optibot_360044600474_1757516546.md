[L1] # Folder Level Permissions

[L2] 

[L3] ### In this article, we'll explain Folder Level Permissions in detail and how to grant desired users and Teams proper access.

[L4] 

[L5] * [Change Folder Permissions](#Permissions)

[L6] 	+ [Restricting or Granting Access by User or Team](#RestrictingGranting)

[L7] 	+ [Folder Security by Team](#ByTeam)

[L10] 

[L11] Folder permission options give account Admins access to granular permissions structuring. It provides many options for controlling and managing who has access to what content.

[L12] 

[L13] Best of all, it can be used on any folder within OptiSigns: Screens, Files/Assets, Playlists, or Schedules.

[L14] 

[L15] Here are some key features:

[L16] 

[L17] * All Owners and Admins can see all folders. Only Users are restricted. See [**Managing User Roles**](https://support.optisigns.com/hc/en-us/articles/360046356113-Advanced-Security-Managing-User-Roles) for more information on User Roles.

[L18] * If you have folders and content set up before activating the Folder Security feature:

[L19] 	+ After activation, all users will still be able to see all folders. You can then tighten security by applying permissions to various folders.

[L20] * Security permissions cannot be set at the asset level

[L21]

[L31] ## Change Folder Permissions

[L32] 

[L33] To apply Folder Security settings, you'll need to create a folder. Folder permissions can be set during folder creation, or after a Folder is made.¬†

[L34] 

[L35] |  |

[L36] | --- |

[L37] | **IMPORTANT** |

[L38] | Anything dealing with Folder Security is specific to users with **Owner**, **Super Admin**, or **Admin** permissions within OptiSigns. These roles are managed by your organization. |

[L39] 

[L40] When creating a **New Folder**, simply click **Advanced** to open up the Folder Security options:

[L52] A few important notes:

[L58] ### Restricting or Granting Access by User or Team

[L59] 

[L60] By default, access is granted to everyone on the Team which created the folder. You will see a greyed out box, which signifies **permissions which cannot be changed**. In this case, because the Default Team is the team which created/is creating this folder, it will need at least some level of access to it.

[L61] 

[L62] We can provide more specific permissions or restrictions, however, by deleting the **Everyone on this team** permission.

[L63] 

[L64] ![new folder everyone on this team x](https://support.optisigns.com/hc/article_attachments/44602802533523)

[L65] 

[L66] Clicking the **blank area** allows you to add a new permission.

[L67] 

[L68] ![new folder permission options](https://support.optisigns.com/hc/article_attachments/44602796071443)

[L69] 

[L70] As you can see, as the account owner or Super Admin, we have a lot of options. We can give access to everyone on the account, everyone on the team, only Team Admins, individual users, or other teams.

[L71] 

[L72] |  |

[L73] | --- |

[L74] | **IMPORTANT** |

[L75] | Users which appear here are ***only users registered on the current team**.* Other users on other teams will not appear. |

[L76] 

[L77] In this case, we'll grant access to our samurai friend Sanjuro here.¬†

[L78] 

[L79] ![new folder added user](https://support.optisigns.com/hc/article_attachments/44602796072595)

[L80] 

[L81] In terms of permissions, this means:

[L82] 

[L83] * All Super Admins have access to this folder;

[L84] * All Team Admins for the Default Team have access to this folder;

[L85] * The User, Kuwabatake Sanjuro, has Edit and View access to this folder;

[L86] 	+ This can be further restricted by unclicking the **Edit** option - this will only allow the User to view the content.

[L87] 	+ Having **Edit** selected but not **View** will allow the User to upload content to the folder, but not see what is inside.

[L88] * No other User, team member, or member of any other team ***will even see*** this folder.

[L132] ## Playlist and Schedule Specific Permissions

[L133] 

[L134] When dealing with Playlists and Schedules, there are some specific nuances.

[L135] 

[L136] #### Playlist-Specific:

[L137] 

[L138] **Example:** A Playlist is created using assets from several folders with various permissions.

[L139] 

[L140] * Anyone with access to all these folders will be able to see all the assets or sub-playlists they can see.

[L141] * Users with only partial access ***will only see the assets or sub-playlists they have access to.***

[L142] * Playlist Total Time will always display the ***full duration,*** including invisible assets or sub-playlists:

[L143] 

[L144] ![](https://support.optisigns.com/hc/article_attachments/44602802557331)

[L145] * A warning message will display for Users stating "There are items in this schedule you do not have access to"

[L146] 

[L147] #### Schedule-Specific:

[L148] 

[L149] **Example:** A screen has a playlist or asset scheduled that is not accessible to all users.

[L150] 

[L151] * Anyone with access to this playlist or asset will be able to see the schedule

[L152] * Users without the proper permissions will see that something is scheduled, but will not be able to select it (grayed-out). It will display as a "Non-accessible item" instead of the name of the asset or playlist.

[L153] * A warning message will display for Users stating "There are items in this schedule that you do not have access to"
