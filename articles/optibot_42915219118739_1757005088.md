[L1] # Legacy DataSources

[L2] 

[L3] ### In this article, we’ll cover our Legacy DataSources - key-value pair formatting with Google Sheets or Microsoft 365 Excel

[L4] 

[L5] * [How it Works and When to Use Legacy DataSources](#HowitWorks)

[L6] * [Creating Data Mapping Elements](#Creating)

[L7] * [Generating a Google Sheet or Excel Document](#Generating)

[L8] * [Modifying the Data](#Modifying)

[L9] 

[L10] Using Designer, it’s possible to assign a key value to an element, then map it to a Google Sheet or Microsoft Excel document. This will create a new spreadsheet directly mapped to the individual design. The values on this new spreadsheet can be modified to change what the design displays.

[L11] 

[L12] ![advanced datasources googlesheet or ms excel 365](https://support.optisigns.com/hc/article_attachments/42915203035795)Please note that this is an older form of Data Mapping in OptiSigns. Now, with our new OptiSync feature, exsiting Google Sheets or MS Excell sheets can be directly integrated into a design with a simple drag-and-drop. It is also possible to directly connect APIs, JSON, XML, or custom tables. See our article on [**Dynamic Data Mapping using OptiSync**](https://support.optisigns.com/hc/en-us/articles/29217646663187-How-to-Set-Up-Dynamic-Data-Mapping-with-OptiSync) for more information.

[L15] 

[L16] ## How it Works and When to Use Legacy DataSources

[L17] 

[L18] This form of Data Mapping follows these steps:

[L19] 

[L20] 1. Mapping data to individual design elements

[L21] 2. Generating a Google or Excel spreadsheet of these elements

[L22] 3. Altering the spreadsheet so changes are reflected on the design

[L23] 

[L24] Different screens can be mapped to the same design so that, depending on where it’s shown, the values will be different.

[L25] 

[L26] Legacy DataSources are particularly useful for pairing with our [**Scrolling Text**](https://support.optisigns.com/hc/en-us/articles/42436941395475-Widgets-in-Designer#ScrollingText) and [**Scrolling Vertical** widgets](https://support.optisigns.com/hc/en-us/articles/42436941395475-Widgets-in-Designer#ScrollingVertical), and if you have numerous elements on a single design you wish to be dynamic.

[L30] ## Creating Data Mapping Elements

[L31] 

[L32] To get started, you’ll need to create a design. To do this, go to **Files/Assets → Apps → Designer**.

[L33] 

[L34] For this example, we’ve used one of our [pre-built Templates](https://canvas.optisigns.com/) as the design. For yours, you can explore our 5000+ template library, or create a design from scratch. See our article on [**Getting Started with Designer**](https://support.optisigns.com/hc/en-us/articles/42087942047379-Getting-Started-with-Designer)if you need help.

[L35] 

[L36] Now, we’ll need to add a Data Mapping to an element. Click the Element you want to map to your DataSource, then hit the **Settings** button. Then, click **Make Data Mapping**.

[L37] 

[L38] ![how to make data mapping](https://support.optisigns.com/hc/article_attachments/42915219044243)

[L39] 

[L40] The element will automatically be given an **Element ID.** We also recommend giving it an **Asset Element Name**, which will appear on your generated spreadsheet and can help identify what the value is.

[L41] 

[L42]  ![data mapping element id and asset element name](https://support.optisigns.com/hc/article_attachments/42915203032083)

[L43] 

[L44] There are also additional options:

[L45] 

[L46] * **Hide when data not available:** When no data is present on the sheet, nothing will display

[L47] * **Empty Data Handling:** When there is no DataSource element, the default behavior is to use the default value. You can adjust this with the following options:

[L48] 	+ **Use Default Value:** Show default content, which is what your element looks like originally.

[L49] 	+ **Use Blank:** Nothing will display.

[L50] 

[L51] For more on the **Advanced Options** here, see [**Property Mapping and Display Format**](https://support.optisigns.com/hc/en-us/articles/29217646663187-How-to-Set-Up-Dynamic-Data-Mapping-with-OptiSync#Property).

[L52] 

[L53] Now, you’ll want to ***repeat this process for every element you wish to be dynamic***. Make sure you do this before you move on to the next step.

[L57] ## Generating a Google Sheet or Excel Document

[L58] 

[L59] Once you’ve mapped all the elements you want to be dynamic, it’s time to generate your spreadsheet. To do this, click the **DataSource** button on the Side Menu, then click **Add DataSource**:

[L60] 

[L61] ![how to add a datasource](https://support.optisigns.com/hc/article_attachments/42915203033491)

[L62] 

[L63] Scroll down in the popup to the **Adv. DataSources** section and choose either **Google Sheets (legacy)** or **MS Excel 365 (legacy)** depending on which you prefer.

[L64] 

[L65] ![advanced datasources googlesheet or ms excel 365](https://support.optisigns.com/hc/article_attachments/42915203035795)

[L66] 

[L67] Clicking one of these will bring up the option to sign in with your account of choice. 

[L68] 

[L69] ![choose between google sheet or ms excel datasource](https://support.optisigns.com/hc/article_attachments/42915203036819)

[L98] ## Modifying the Data

[L99] 

[L100] To change the mapping, simply edit the values in the columns, being sure to keep the Asset Element ID and Asset Element Name the same. 

[L101] 

[L102] For example, say we want the word “SOCIAL” to appear on one screen in one location, but we want it to say “COMMUNAL” in another. To do this, we simply duplicate the Row, then change the “Screen Name” and “Value” cells:

[L103] 

[L104] ![modified data in datasource](https://support.optisigns.com/hc/article_attachments/42915203045779)

[L105] 

[L106] The “Screen Name” will need to be the same as the name of the screen you’re targeting with the asset. This is equivalent to the **Device Name** under the **Edit Screen** tab:

[L107] 

[L108] ![device name on edit screen tab](https://support.optisigns.com/hc/article_attachments/42915203048467)

[L109] 

[L110] Some best practices:

[L111] 

[L112] * OptiSigns will identify the screens by name and adjust the Value based on what's entered on your spreadsheet.

[L113] * If you change your screen's name, be sure to update your spreadsheet at the same time. If updated later, it could cause issues with data mapping on the screen at the next update interval.

[L114] * \*\*\*ALL\*\*\* tells OptiSigns that if a screen is not specifically assigned values, it will take value from this row of data. This is equivalent to the "Default Value" mentioned earlier in the article.

[L115] 	+ If a value is detected, it will override the \*\*\*ALL\*\*\* value, like in the example above.
