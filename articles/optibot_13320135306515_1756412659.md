[L1] # Build Integration Using OptiSigns Enterprise App

[L2] 

[L3] The digital signage use case for enterprise user is complex. There may be needs to integrate with various systems and have those systems to initiate a process to send data and control the screen.Or in some cases, there may be a need even to build a custom single purpose app used to operate the screens. OptiSigns Enterprise App is specifically designed for these types of real-time enterprise data integrations. With OptiSigns Enterprise app, you can easily achieve:

[L4] 

[L5] * Load a custom-designed page on the screen as needed and have data points integrated with any applications that need to control the screen.

[L6] * Embed and build the integration with your enterprise applications and allow you to send freeform data to the screens.

[L7] * Develop your own single purpose application to manage the screens and assets with all the APIs that are available with the framework.

[L8] 

[L9] OptiSigns Enterprise app is a framework and pre-built template to help enterprise customers easily build integration with their enterprise applications that need to send data to the screen and control the screen. It can be used in various cases, for example, you can use it to send customer specific communications from your CS system to your customers. In this guide, we will walk you through how to use the OptiSigns Enterprise app with the example below.  

[L10] ![mceclip0.png](https://support.optisigns.com/hc/article_attachments/13440210713235)

[L11] 

[L12] ### **How to use OptiSigns Enterprise App:**

[L13] 

[L14] OptiSigns Enterprise App is more than a framework, it also contains templates to help you with the development and deployment. There are mainly 3 steps to get Enterprise App setup and working properly.

[L15] 

[L16] 1. Create Enterprise App and build your custom designed page.

[L17] 2. Build the integration to your enterprise systems or custom applications. For this tutorial, we will use the demo control center as an example.

[L18] 3. Send data from your system to the screen and get it published.

[L19] 

[L20] #### **1. Create Enterprise App and build custom designed page.**

[L21] 

[L22] Go to┬Ā[app.optisigns.com](https://app.optisigns.com/).

[L23] 

[L24] Click Files/Assets, then click Apps, and select "Enterprise App".

[L25] 

[L26] ![mceclip1.png](https://support.optisigns.com/hc/article_attachments/13320712580627)

[L27] 

[L28] In the next page, click the template picture to create new project, this will be launching a WebIDE with sample code, where you can make adjustment to customize the page layout and how you want to interact with the data points from other systems. If you have an existing URL available, you can use it as well.  

[L29] ![mceclip3.png](https://support.optisigns.com/hc/article_attachments/13320774310291)

[L30] 

[L31] The template comes with sample code that you can use directly, all you need to do is to change how you would like the page to look like and how you want to handle the incoming data points from other systems. Once you are done with the changes, click "Save&Deploy" button, it will be ready for use.

[L32] 

[L33] ![mceclip4.png](https://support.optisigns.com/hc/article_attachments/13320838658451)

[L34] 

[L35] ┬Ā

[L36] 

[L37] #### **2. Build the integration to your enterprise systems or custom applications.**

[L38] 

[L39] We will use the demo Control Center to show how you can send data and interact with the screens in this tutorial. In your cases, the demo Control Center will be your systems that you want to initiate the process to control the screen or a custom application that was specifically built for your need.

[L40] 

[L41] Click "Control Center" in the Enterprise App, it will launch the demo Control Center.┬Ā┬Ā

[L42] 

[L43] ![mceclip5.png](https://support.optisigns.com/hc/article_attachments/13320889712915)

[L44] 

[L45] The demo Control Center will launch with the logon page first. In this case, you just need to use your OptiSigns account. The framework supports authentication and authorization, when you build your integrations with other systems, you will have the capability to utilize the authorization you defined with in OptiSigns.┬Ā

[L46] 

[L47] After logon, the demo Control Center will list all the screens that you have access to. You can also search the screens that you want to control. In this case, we will try to control the Enterprise App Demo Screen.

[L48] 

[L49] ![mceclip7.png](https://support.optisigns.com/hc/article_attachments/13321031641363)

[L50] 

[L51] Click the "Edit" button will popup the Edit Device page. This is where we can control how to send data to the screen.

[L52] 

[L53] ![mceclip8.png](https://support.optisigns.com/hc/article_attachments/13321033226131)

[L54] 

[L55] #### **3. Send data from your system to the screen and get it published.**

[L56] 

[L57] Looking at this screen, we can have any asset/playlist/schedule assigned to the screen. The screen will just play the assigned content as normal. When we initiate the integration from the demo Control Center, we can send the freeform JSON data to the screen, and assign the Enterprise App asset ID to take over the screen immediately and display the data.

[L58] 

[L59] Now, the screen is assigned an asset other than the Enterprise App.

[L60] 

[L61] ![mceclip9.png](https://support.optisigns.com/hc/article_attachments/13321076982035)

[L62] 

[L63] If we go back to the demo Control Center, and put in the data that we want to send to the screen, and the Enterprise App asset id, then click Save. It will take over the screen and load the Enterprise App on the screen with the data that we sent over.  

[L64] Once done, you can click the "Stop" button to end the integration.

[L65] 

[L66] ![mceclip2.png](https://support.optisigns.com/hc/article_attachments/13440341376531)

[L67] 

[L68] To verify the data that was sent to the screen, you can go back to the OptiSigns portal, find the screen. Expand the "Advanced" section, then expand "More" section.┬Ā

[L69] 

[L70] At the bottom of the Edit Screen page, click the button to show the Device Specific Info. You can see the same data sent from the demo Control Center.

[L71] 

[L72] ![mceclip3.png](https://support.optisigns.com/hc/article_attachments/13440441484051)

[L73] 

[L74] On the screen, you will see those data sent from the demo Control Center are displayed accordingly. Including the customer name, car model, advisor name, and the image.

[L75] 

[L76] ![mceclip4.png](https://support.optisigns.com/hc/article_attachments/13440515661203)

[L77]
