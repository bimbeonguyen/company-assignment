[L100] ### Advanced Settings

[L101] 

[L102] ![](https://support.optisigns.com/hc/article_attachments/44646006336915)

[L103] 

[L104] On the **Advanced Settings** screen, you can perform various functions such as enabling/disabling Network Proxy, NTP, the On-screen Keyboard, SSH, configuring a Static IP for either WLAN or Ethernet, and enabling 802.1x Ethernet. [L108] * **Certificate File** **-** Allows you to [install a root certificate](https://support.optisigns.com/hc/en-us/articles/35184720136595-How-to-Install-a-Root-Certificate-and-Display-an-Internal-Website-on-Screens) for displaying an internal website. [L109] * **NTP Server -** Input server information for your Network Time Protocol (NTP). This can be used to ensure the OptiSigns Pro Player has its computer clock time with other time sources in your network.

[L111] * **Network Proxy -** Server information for any network proxy. This is a system or router which serves as a middleman between our Pro Player and the internet and can be used to enhance security, privacy, or efficiency.

[L113] * **Static IP Information -** An IP address that remains the same over time. We allow two types of Static IP: WLAN or ETH. These can improve network performance. This screenshot shows WLAN; if Static ETH IP is chosen, these will be ETH IP Address, Default Gateway, Subnet Mask, and DNS Server.

[L114] 	+ **IP Address -**Set the Static IP address.

[L115] 	+ **Default Gateway**- Set the Default Gateway for the Static IP address. This is the IP address of your router.

[L116] 	+ **Subnet Mask**- Set the Subnet Mask for the Static IP. Usually, this number is 255.255.255.0 or some variant of this.

[L117] 	+ **DNS Server**- Lets you set up your DNS server for the Static IP address.

[L118] 

[L119] ![](https://support.optisigns.com/hc/article_attachments/44646006341779)

[L120] 

[L121] * **802.1x UserName**- Username of the 802.1x Enterprise Ethernet network

[L122] * **802.1x Password**- Password of the 802.1x Enterprise Ethernet network

[L123] * **802.1x Certificate**- Allows you to install a certificate for accessing an 802.1x Enterprise network. The certificate will need to be present locally on the device in order to be installed.

[L157] ## How to Use SSH to Remote Into Device

[L158] 

[L159] Pro Players allow remoting into devices using SSH. Here's how to set that up.

[L160] 

[L161] First, enable **SSH**in your Advanced Settings.

[L162] 

[L163] ![ssh advanced settings](https://support.optisigns.com/hc/article_attachments/40985616289939)

[L164] 

[L165] This will provide you with the SSH IP and Port number. By default, the port is **3000**, but it can be changed to whatever you like.

[L166] 

[L167] ![ssh ip and port](https://support.optisigns.com/hc/article_attachments/40985596548243)

[L168] 

[L169] Now that SSH is enabled and you have the IP and Port, you can use a computer terminal to remote into the device.

[L170] 

[L171] Type the following command:

[L172] 

[L173] ```

[L174] SSH [optisigns@<ip-address-here>](mailto:optisigns@<ip-address-here>) -p <port-number-here>

[L175] ```

[L176] When it asks for a password, type **optisigns**. This should connect you to the device via SSH.

[L177] 

[L178] To change the default password (which we ***highly┬Ārecommend***), type:

[L179] 

[L180] ```

[L181] passwd

[L182] ```

[L183] This will ask you to type the current password, then new, then to type in the new password again.

[L184] 

[L185] ![](https://support.optisigns.com/hc/article_attachments/41021980486931)

[L186] 

[L187] |  |

[L188] | --- |

[L189] | **NOTE** |

[L190] | Restarting the OptiSigns Pro Player will cause SSH to automatically disable for security purposes. |

[L191] 

[L192] That's it! You should be able to use the SSH function now.
