[L1] # Pagination

[L2] 

[L3] GraphQL enables you to fetch only the fields you need. This helps keep network responses small and fast. However, GraphQL doesn't automatically guarantee small responses. If the result list contains a lot of elements, it could cause an enormous response. You will need to do pagination in this situation. 

[L4] 

[L5] Our GraphQL API implemented cursor-based pagination. There are 4 parameters used to traverse through the elements.

[L6] 

[L7] * after: Paginate after opaque cursor

[L8] * before: Paginate before opaque cursor

[L9] * first: Paginate first x elements

[L10] * last: Paginate last x elements

[L11] 

[L12] In the query, you can request the pageInfo and pageData object, which will provide all the needed data for pagination. Below is an example to paginate on devices response. It requests the first element after the specified cursor.

[L13] 

[L14] ```

[L15] query{  

[L16]   devices (query : {},first:1,after:"YXJyYXljb25uZWN0aW9uOjQ=") {  

[L17]     page{  

[L18]       edges{  

[L19]         cursor,  

[L20]         node{  

[L21]           _id,  

[L22]           deviceName,  

[L23]           UUID,  

[L24]           pairingCode,  

[L25]     			currentType,  

[L26]     			currentAssetId,  

[L27]           currentPlaylistId,  

[L28]           localAppVersion,  

[L29]         },  

[L30]       },  

[L31]       pageInfo{  

[L32]         hasNextPage,  

[L33]         startCursor,  

[L34]         endCursor  

[L35]       }  

[L36]     },  

[L37]     pageData{  

[L38]       limit,  

[L39]       offset  

[L40]     }  

[L41]   }  

[L42] }

[L43] ```
