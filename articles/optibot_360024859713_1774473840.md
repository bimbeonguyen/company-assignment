[L99] * **Name -** Name of your Power BI app instance. This is the name of the app in your asset list. It will **not**┬Ābe displayed on your screens.

[L104] * **Direct Login -** In order to view your dashboard on any screen, OptiSigns requires you to authenticate via the pass-through to Microsoft's Power BI service. Simply input your Microsoft ID and password.

[L124] ## Filtering a Report

[L125] 

[L126] If you only wish to display certain pieces of data on your report, you can create Filters within the Power BI app.

[L127] 

[L128] ### Creating a Basic Filter

[L129] 

[L130] For a basic filter, create a Power BI app or open an existing one and open the **Advanced Settings** at the bottom.

[L131] 

[L132] ![](https://support.optisigns.com/hc/article_attachments/44002355213715)

[L133] 

[L134] To create a filter, weŌĆÖll need three key pieces of information from your report: a **Table**, a **Column**, and a **Value**.

[L135] 

[L136] ![](https://support.optisigns.com/hc/article_attachments/44002377534611)

[L137] 

[L138] To find these, go to the report you wish to use. Click **Edit**.

[L139] 

[L140] ![](https://support.optisigns.com/hc/article_attachments/44002377537299)

[L141] 

[L142] A number of options will appear on the right side. Under the **Data** tab, you should see a number of folders and subfolders. The folders correspond to the **Table**, the subfolder the **Column**. When selected, there will be a number of names in a properly configured report in the **Filters** column. These are your **Values**. See the image below for a better visual reference.

[L143] 

[L144] ![](https://support.optisigns.com/hc/article_attachments/44018968307347)

[L145] 

[L146] In this example, weŌĆÖll use **Account** as our Table, **Region** as our Column, and **Central** as our filter Value. Simply fill that in to the Filter.

[L147] 

[L148] ![](https://support.optisigns.com/hc/article_attachments/44002377545747)

[L149] 

[L150] There is one last step. We need to set our variable, which corresponds to the field between the Column and the Value. This is clickable, and there are many options:

[L151] 

[L152] ![](https://support.optisigns.com/hc/article_attachments/44002377551635)

[L153] 

[L154] This gives you a great degree of customizability on how you wish to set up your Filter. For our example, weŌĆÖll keep the value as **Is**. This means that it will only show data corresponding to that Table, Column, and Value.

[L155] 

[L156] If this is all you need, great. YouŌĆÖre done! Simply hit **Save** and your filter will apply. However, itŌĆÖs also possible to add additional conditions to a filter, or to create more than one per report.

[L157] 

[L158] To add additional conditions to a basic filter, hit the **+** button next to it.

[L159] 

[L160] ![](https://support.optisigns.com/hc/article_attachments/44002377555603)

[L161] 

[L162] You can change the condition logic between **AND** and **OR** to specify what type of filter to apply. Then, set the variable and value. You can continue adding additional conditions if you wish.

[L163] 

[L164] To add a completely new filter, hit **Add New Filter**.

[L165] 

[L166] ![](https://support.optisigns.com/hc/article_attachments/44002355238035)

[L167] 

[L168] This will (shockingly) create a new filter. Fill this out as you did your first one, with the information you wish to show.

[L169] 

[L170] ![](https://support.optisigns.com/hc/article_attachments/44002377558675)

[L171] 

[L172] ### Creating a Filter to Automatically Target Certain Screens (Optional)

[L173] 

[L174] By pairing these filters with OptiSigns [**Device Additional Attributes**](https://support.optisigns.com/hc/en-us/articles/360048914673-Edit-Screen-What-does-each-option-do#attributes), it is possible to apply them only to certain screens. This is useful if you have multiple screen locations, for example, and only wish to show Power BI data which is relevant to them.

[L175] 

[L176] To set this up, navigate to the **Device Additional Attributes** by editing your screen. This can be found through the **Screens tab,** then finding the screen you wish to Edit. Click **Edit Screen ŌåÆ Advanced ŌåÆ More ŌåÆ Device Additional Attributes**.  

[L177] ![](https://support.optisigns.com/hc/article_attachments/44002355241875)

[L178] 

[L179] HereŌĆÖs where it gets fun. On the Device Additional Attributes screen, youŌĆÖll see two fields: **Key**, and **Value**.

[L180] 

[L181] ![](https://support.optisigns.com/hc/article_attachments/44002377564179)

[L182] 

[L183] * **Key** - A parameter that will be used by the filter. This will replace either the Table, Column, or Value, as youŌĆÖll see in a moment. YouŌĆÖll want to keep your Key consistent across ALL screens where you plan to display a filtered Power BI report.

[L184] * **Value** - Dictates which part of the report to share with this screen. YouŌĆÖll want this to vary depending on the screen.

[L185] 

[L186] For this example, we will fill in the Key as **Location** and the Value as **Central**:

[L187] 

[L188] ![](https://support.optisigns.com/hc/article_attachments/44002355249939)

[L189] 

[L190] For practical purposes, what weŌĆÖre saying here is that this screenŌĆÖs Location is in the Central region, which corresponds to the Values which exist on our Power BI report. You can add as many attributes to an individual screen as you wish.

[L191] 

[L192] |  |

[L193] | --- |

[L194] | **IMPORTANT** |

[L195] | The Value here MUST match up with an element of your report if you wish to apply the filter properly. In our example report, we have Central, East, and West, so one of these must be the value for the report to display properly. Your report will be different. |

[L196] 

[L197] Now that weŌĆÖve set this up, we can return to our Power BI report. Now, weŌĆÖll substitute the **Value** for **{{Location}}**:

[L198] 

[L199] ![](https://support.optisigns.com/hc/article_attachments/44002377569939)

[L200] 

[L201] By inputting this and assigning this to a screen, it will find the Device Additional Attribute and substitute the Value here. In this case, that value is Central, so it will filter out all data that does not fall under the Account Table, Region Column, and Central Value.┬Ā

[L202] 

[L203] For a different screen, you might set the Device Additional Attribute value to East. By pairing this same report to that screen, it will filter out all data that does not fall under the Account Table, Region Column, and East value.

[L204] 

[L205] LetŌĆÖs see how this works with a practical example:

[L206] 

[L207] * Say we have 3 screens in 3 locations:

[L208] 	+ Screen A is in Location 1, Screen B is in Location 2, and Screen C is in Location 3

[L209] * We only want these screens to show the appropriate data off this report

[L210] 	+ On each screen, we go to Device Additional Attributes. We set Key as Location and the Value as Location 1 for Screen A; Location 2 for B; and Location 3 for C

[L211] 	+ We set the Value in our Power BI report to {{Location}}

[L212] * It will automatically filter the report based on the deviceŌĆÖs set location, and this app can be used across all 3 screens and will show different data depending on where the screen is located

[L213] 

[L214] Pretty cool, huh? This can also be used to replace the Table or Column values depending on your use case or need.

[L242] #### **How does security work with OptiSigns Power BI integration?**

[L243] 

[L244] OptiSigns integrates with and displays Power BI dashboards via an official Microsoft API, securely integrated through your Power BI or MS Azure portal. No usernames or passwords are stored in OptiSigns.

[L245] 

[L246] Your devices (screens) will display your Power BI report on the screens directly. Power BI data does not pass through our servers. There is no data-farming on our end of any kind.

[L247] 

[L248] OptiSigns uses Microsoft APIs for integration. In order for our integrations to work, the integration has to be approved by an administrator. This is the same across all integrations using Microsoft APIs.

[L249] 

[L250] This administrator access is only needed for first time access. Once the OptiSigns app is approved for use, other users can use OptiSigns directly.

[L251] 

[L252] Customers with MS Azure Enterprise Apps management can also [manage OptiSigns in your Enterprise App](https://support.optisigns.com/hc/en-us/articles/4403616315539) for even more control over security options.

[L274] #### **I'm having trouble displaying my Power BI report off MS Fabric.**

[L282] #### **My Power BI report displays on my portal, but won't display on my screen. Help?**

[L283] 

[L284] This issue is usually caused by network connectivity issues at the device. We recommend checking and validating your device/screen's network connection.

[L285] 

[L286] If you have a Samsung SSSP or LG WebOS TV, there might be a different issue. Microsoft requires a minimum Chromium version of 95 for their apps to display. These TVs typically don't update very often. This is a Microsoft issue, and there is little we can do on our end.

[L287] 

[L288] If you have one of these TVs and wish to display SharePoint, we recommend our [Android Player](https://www.optisigns.com/product/hardware/android-player).

[L289] 

[L290] If you're still having issues, feel free to contact our support team at [support@optisigns.com](mailto:support@optisigns.com).

[L292] #### **My Power BI report lags and crashes frequently. Why?**

[L293] 

[L294] Lagging and crashing Power BI reports usually have to do with two factors:

[L295] 

[L296] * The device being used to run the OptiSigns app and display the report

[L297] * The size of the report

[L298] 

[L299] Simply put, the larger the size of the report, the more powerful device you'll need to display it without issue. Small reports can use weaker hardware, while large and sprawling reports will need more powerful or dedicated hardware.

[L300] 

[L301] This means, if you're having this issue, you'll need to either:

[L302] 

[L303] 1. Reduce the size of the report you want to display

[L304] 2. Improve the hardware you're using

[L305] 

[L306] We recommend using an [**OptiSigns Pro Player**](https://www.optisigns.com/product/hardware/pro-digital-signage-player)or [**OptiSigns ProMax Player**](https://www.optisigns.com/product/hardware/promax-digital-signage-player)for displaying large, heavy Power BI reports.

[L307] 

[L308] #### **Power BI Display for US Government Customers**

[L309] 

[L310] If you are a US government entity (federal, state, or local), you may be using Power BI for government. If so, your Power BI reports will use one of these URLs:

[L311] 

[L312] * **GCC**: `https://app.powerbigov.us`

[L313] * **GCC High**: [`https://app.high.powerbigov.us`](https://app.high.powerbigov.us)

[L315] If that is the case, your display won't work with the Power BI app. However, it is possible to create your Power BI as a Sharepoint application, then display it with our [**SharePoint app**](https://support.optisigns.com/hc/en-us/articles/4414539282067-Displaying-SharePoint-Sites-on-OptiSigns)**.**
