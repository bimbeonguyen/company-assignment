[L16] ## What is Web Scripting?

[L17] 

[L18] Web Scripting is a feature that allows users to automate navigation and form entry on websites without any coding. This tool records user actions, such as entering usernames and passwords, navigating to specific pages, handling pop-ups, running your own JavaScript and more on a website, and then encrypts the recorded script for secure execution on designated devices.

[L19] 

[L20] OptiSigns encrypts all the scripts and your password entered with our own private keys.

[L21] 

[L22] * This will ensure your data/password is safe as soon as it left your browsers and only get decrypted at device level before the fields are entered.

[L23] 

[L24] We also provide Zero Knowledge encryption method so that you can protect your script with your own Master Password. *You can read more at the end of this article.*

[L25] 

[L26] If your dashboard requires login with the 2FA, OptiSigns supports the 2FA in the Web Scripting app. You can read more [here](https://support.optisigns.com/hc/en-us/articles/19145077187859).

[L143] ## **Create Web Scripting Asset in OptiSigns**

[L144] 

[L145] Log in to OptiSigns <https://app.optisigns.com/>

[L146] 

[L147] Click **File/Assets**

[L148] 

[L149] Click **Apps** and select **Web Scripting.**

[L150] 

[L151] ![](https://support.optisigns.com/hc/article_attachments/51710297906323)

[L152] 

[L153] Enter the information for your Web Scripting asset:

[L154] 

[L155] ![](https://support.optisigns.com/hc/article_attachments/51710297909011)

[L156] 

[L157] * **Name:** Name of your asset your asset list. It will **not** be displayed on your screens.

[L158] * **Master Password:** By default, OptiSigns will encrypt the whole script with OptiSigns private key to protect your script, especially username, password in the script. You can add another protection layer by entering a Master Password. If you enter Master Password here, at each device, you will need to enter that Master Password one time in OptiSigns app so it can decrypt the content.

[L159] * **Recorded Script:** Paste the script recorded by the Navigation Recorder here. You can take notice to the script, it's actually not very complicated, you can make some minor changes if you need to.

[L160] * **Scripts Injection:** Allows you to add specific scripts into your web script.

[L161] * **Block third-party cookies:** Has the web script block third-party cookies.

[L162] 

[L163] ![web scripting app section 2](https://support.optisigns.com/hc/article_attachments/51710297911443)

[L164] * **Secret 2FA:** This is the Secret Key mentioned in the 2FA section. Only needed if your login requires 2FA.

[L165] * **Recorded 2FA Code:** If your login requires 2FA, this is where you input the code you received. Paired with the Secret Key, this keeps your 2FA channel open for this asset to use repeatedly.

[L166] * **Delay Execute 2FA JavaScript:** Delays execution of JavaScript elements on the 2FA element by a set amount of time, measured in seconds.

[L167] * **Delay Executing JavaScript:** Delays execution of JavaScript elements on the page by a set amount of time, measured in seconds.

[L171] ## **Deploying Your Web-Scripting Asset on Your Screens**

[L172] 

[L173] Once created, the Web Scripting asset can be used in by itself or in Playlists, Schedules, or SplitScreen zones just like any other assets.

[L174] 

[L175] If you don't use Master Password, device will use OptiSigns private key to decrypt and execute, so no additional action is needed on the devices.

[L176] 

[L177] If you choose to use Master Password and our Zero Knowledge Encryption framework, you will have to enter your Master Password at each device

[L178] 

[L179] * This only needs to be done once on each device and can be used to decrypt all Scripting Assets. (Of course, you have to use the same Master Password to encrypt them).

[L183] ## **Web Scripting Security Features**

[L184] 

[L185] When anyone other than the creator of a Web Scripting asset looks at it, it will look like this:

[L186] 

[L187] ![web scripting security other owner](https://support.optisigns.com/hc/article_attachments/51710297912979)

[L188] 

[L189] Sensitive information will be hidden, and the **Override** button must be hit before the Recorded Script or Injection can be altered. There is no way for any non-owner to actually see what the script says.

[L190] 

[L191] If a **Master Password** is enabled, the owner will have to enter a password before the script is revealed.

[L192] 

[L193] **Here's how the encryption flow works:**

[L194] 

[L195] ![mceclip12.png](https://support.optisigns.com/hc/article_attachments/1500019937601)

[L196] 

[L197] If you want to add additional security by utilizing a Master Password and our Zero Knowledge Encryption framework you will have to enter your Master Password when:

[L198] 

[L199] * Creating/editing assets

[L200] * One time at each devices, so it will know how to decrypt

[L201] 

[L202] Your script is encrypted at your browser, and transfer securely using HTTPS/SSL during transits and stored on our servers.  

[L203] It then sends securely to devices, and decrypted at device level right before executing on the target website.
