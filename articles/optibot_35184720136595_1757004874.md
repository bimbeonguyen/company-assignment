[L21] OptiSigns digital signage software is an extremely valuable tool for internal communication, capable of displaying internal memos, communications, and announcements across numerous locations and offices. Messaging can be tailored to specific audiences, scheduled, and automatically synced to data sources.

[L22] 

[L23] With all these capabilities, it stands to reason that some want to incorporate OptiSigns into their intranet, or internal website. These can be displayed on your screens using the OptiSigns Website app, or Advanced Website app in some cases.

[L24] 

[L25] However, in order to do this, you’ll need a trusted, self-signed root certificate installed on the platform you choose to run it on. Here, we’ll show you how to get that installed.

[L26] 

[L27] Please note that you’ll need to obtain a trusted certificate yourself. This will need to be some sort of key, usually denoted by a **.pem** file extension.

[L28] 

[L29] In this article, we will walk you through two main steps:

[L30] 

[L31] 1. Installing a root certificate on your device of choice (OptiSigns Pro Player, Ubuntu, Windows, MacOS, or Chrome browser)

[L32] 2. Configuring a Website app to display your internal website

[L33] 

[L34]   | **NOTE** |

[L35] | Before you start, make sure you’re connected to the same network as the server you’re trying to access. |

[L36] 

[L37] |  |

[L38] | --- |

[L39] | **Please note that Android devices are not supported at this time.** |

[L43] ## Installing a Root Certificate on an OptiSigns Gen3 Pro Player

[L44] 

[L45] In order to get your root certificate onto an OptiSigns Pro Digital Signage Player, you’ll need to manually add it. Find where your certificate is stored, then download it to a USB or MicroSD card. Plug the card into the Pro Player.

[L46] 

[L47]   | **IMPORTANT** |

[L48] | For installation on a Gen 3 Pro Player, your certificate must have a **.crt** extension. However, it is important that this certificate is signed and contains your public key. These are usually generated as **.pem** files. You’ll need to rename your certificate (.pem) file and change its extension to **.crt** for your internal website to properly display. |

[L49] 

[L50] Now, open your OptiSigns player menu. Go to **About → Advanced Settings**.

[L51] 

[L52] **![](https://support.optisigns.com/hc/article_attachments/35184705322515)**

[L53] 

[L54] Here, you’ll see a field called **Certificate File**. 

[L55] 

[L56] ![](https://support.optisigns.com/hc/article_attachments/35184705339539)

[L57] 

[L58] Simply locate your certificate on your USB or MicroSD card and select it. The certificate will automatically be downloaded to the appropriate location. This will allow your OptiSigns player to display your internal website.

[L62] ## Installing a Root Certificate on OptiSigns Gen 2 Pro Players and Linux/Ubuntu Devices

[L63] 

[L64] To install a root certificate on a Linux or Ubuntu device, you’ll need to make heavy use of the **Terminal.**

[L65] 

[L66] To begin, take your trusted, signed certificate (.pem file) and place it in the /usr/share/ca-certificates folder.

[L67] 

[L68] ![](https://support.optisigns.com/hc/article_attachments/35184720058515)

[L69] 

[L70]   | **NOTE** |

[L71] | You will need to rename your **.pem** file to make it a **.crt** file, or else this will not work. |

[L72] 

[L73] After the certificate has been moved and renamed, you’ll need to refresh the installed certificates and hashes. Open your **Terminal** and type in this command:

[L74] 

[L75] ```

[L76] sudo update-ca-certificates

[L77] ```

[L78] 

[L79] Once this command is executed, it should say that it has installed 1 (or more) new certificate.

[L80] 

[L81] ![](https://support.optisigns.com/hc/article_attachments/35184720067475)

[L82] 

[L83] This means the certificate has been added to the operating system and signed certificates will be trusted.

[L84] 

[L85] Now, you’ll want to install the certificate in the Chromium store using this command:

[L86] 

[L87] ```

[L88] certutil -A -n "ROOT-CA" -t "TCu,Cu,Tu" -i /usr/share/ca-certificates/[name-of-cert].crt -d sql:/home/[user]/.pki/nssdb

[L89] ```

[L90] 

[L91] The Linux-based OptiSigns app uses Chromium, so this will allow the certificate to pass through the OptiSigns app.

[L92] 

[L93] Now reboot your device.

[L94] 

[L95] Congratulations! You’ve successfully installed a root certificate on Ubuntu.

[L99] ## Installing a Root Certificate on a Windows Device

[L100] 

[L101] Broadly, there are four major steps to installing a root certificate locally on a Windows device:

[L102] 

[L103] 1. Open the Microsoft Management Console (MMC)

[L104] 2. Add the Certificates Snap-In

[L105] 3. Import the Certificate

[L106] 4. Verify the Installation

[L107] 

[L108] These same steps can be performed quickly using the Windows Terminal, if you’re so inclined.

[L109] 

[L110] Be cautious when installing root certificates, especially self-signed ones. Only install certificates from sources you trust completely.

[L111] 

[L112]   | **NOTE** |

[L113] | You need administrator privileges to install certificates in the Trusted Root Certification Authorities store. |

[L114] 

[L115] ### Open the Microsoft Management Console (MMC)

[L116] 

[L117] The first step is opening the Microsoft Management Console (MMC), which is a framework for managing Windows components.

[L118] 

[L119] To do this, click **Start,** type **mmc** into the Search bar, then press **Enter**.

[L120] 

[L121] ![](https://support.optisigns.com/hc/article_attachments/35184705345939)

[L122] 

[L123] ### Add the Certificates Snap-In

[L124] 

[L125] With the MMC open, go to **File → Add/Remove Snap-In**.

[L126] 

[L127] ![](https://support.optisigns.com/hc/article_attachments/35184705348371)

[L128] 

[L129] In the “Available snap-ins” list, select **Certificates** and click **Add.**

[L130] 

[L131] ![](https://support.optisigns.com/hc/article_attachments/35184720078227)

[L132] 

[L133] Choose **Computer account** to manage certificates for the local computer. Click **Next**.

[L134] 

[L135] ![](https://support.optisigns.com/hc/article_attachments/35184705352851)

[L136] 

[L137] Next, click **Local computer** and click **Finish**.

[L138] 

[L139] ![](https://support.optisigns.com/hc/article_attachments/35184705355155)

[L140] 

[L141] Click **OK** to close the “Add or Remove Snap-ins” dialog.

[L142] 

[L143] ### Import the Certificate

[L144] 

[L145] Now **Certificates (Local Computer)** will appear as an option in your MMC. To continue, exp[and the **Certificates (Local Computer)** option within the right pane of the console. Then, expand **Trusted Root Certification Authorities**.

[L146] 

[L147] ![](https://support.optisigns.com/hc/article_attachments/35184720086931)

[L148] 

[L149] Right-click on the **Certificates** folder and select **All Tasks → Import**.

[L150] 

[L151] **![](https://support.optisigns.com/hc/article_attachments/35184720091795)**

[L152] 

[L153] This will open the Certificate Import Wizard. Click **Next** to get started.

[L154] 

[L155] Now, click **Browse.** Locate your self-signed certificate and select it. This should be a .pem file. You may need to expand the file name options to **All Files** to see it.

[L156] 

[L157] ![](https://support.optisigns.com/hc/article_attachments/35184705362323)

[L158] 

[L159] Select the file and click **Open → Next.**

[L160] 

[L161] Select **Place all certificates in the following store.** Ensure that **Trusted Root Certification Authorities** is selected, then click **Next.**

[L162] 

[L163] **![](https://support.optisigns.com/hc/article_attachments/35184705364115)**

[L164] 

[L165] Now you’ll be asked to confirm whether you want to complete the import. As long as you trust the certificate, agree to it by hitting **Finish**.

[L166] 

[L167] **![](https://support.optisigns.com/hc/article_attachments/35184705368339)**

[L168] 

[L169] ### Verify the Installation

[L170] 

[L171] Close the MMC.

[L172] 

[L173] Open a web browser and try accessing the local host domain. If the certificate is installed correctly, you should no longer see the security warning. However, because it's a self-signed root certificate, you might still see a warning indicating that the website's identity cannot be verified.

[L174] 

[L175] If you encounter any issues, double-check the file paths, file names, and the output of the Certificate Import Wizard for any error messages.

[L176] 

[L177] ### Command Line Installation

[L178] 

[L179] You can also use the **Terminal** to directly install the certificate. Simply open up the Terminal app with Administrator privileges after searching for it on your Start bar, and input the following command:

[L180] 

[L181] ```

[L182] certutil –addstore –f "Root" “C:\path\to\your\_certificate\_file”

[L183] ```

[L184] 

[L185] This will automatically install the certificate to the Trusted Root Certificates folder and you should be able to access the host domain in the same way as above.

[L189] ## Installing a Root Certificate on a MacOS Device

[L190] 

[L191] To prepare for the installation, make sure your device is connected to the same network of host servers you plan to use. Also, make sure your certificate is in a folder (the Download folder will work) on the device installing the certificates.

[L192] 

[L193] ### Install Certificate

[L194] 

[L195] To begin, open **Keychain Access**. This is normally located in the “Other” folder in the launchpad.

[L196] 

[L197] ![](https://support.optisigns.com/hc/article_attachments/35184720101907)

[L198] 

[L199] Select the System tab within the menu on the left. If you see a padlock icon next to the System folder, right click to unlock and enter the system password. 

[L200] 

[L201] ![](https://support.optisigns.com/hc/article_attachments/35184720103571)

[L202] 

[L203] ![](https://support.optisigns.com/hc/article_attachments/35184720107027)

[L204] 

[L205] Open the folder where your certificate is stored. Drag and drop the certificate into the system folder in Keychain Access. If a red x is displayed next to the certificate like below, keep following along. Otherwise, you’re done.

[L206] 

[L207] ![](https://support.optisigns.com/hc/article_attachments/35184705380883)

[L208] 

[L209] Right click the certificate and select “get info”

[L210] 

[L211] ![](https://support.optisigns.com/hc/article_attachments/35184705382291)

[L212] 

[L213] Select “Trust”.

[L214] 

[L215] ![](https://support.optisigns.com/hc/article_attachments/35184720114579)

[L216] 

[L217] Select “Always Trust”. This means your computer will always trust this certificate to keep your connection secure.

[L218] 

[L219] ![](https://support.optisigns.com/hc/article_attachments/35184705386771)

[L220] 

[L221] Exit and you will be prompted with entering password. Enter the system password.

[L222] 

[L223] Your certificate is now installed. You will now be able to access the local website. 

[L224] 

[L225] ### Command Line Installation

[L226] 

[L227] On MacOS, you can also use the Terminal to directly install the Certificate. Simply type in these commands:

[L228] 

[L229] **Use the following command to add a certificate:**

[L230] 

[L231] ```

[L232] sudo security add-trusted-cert -d -r trustRoot -k /Library/Keychains/System.keychain “new-root-certificate”

[L233] ```

[L234] 

[L235] **Use the following command to remove a certificate:**

[L236] 

[L237] ```

[L238] sudo security delete-certificate -c "name of existing certificate"

[L239] ```

[L243] ## Installing a Root Certificate on Chrome-Based Web Browsers

[L244] 

[L245] Depending on the operating system, Chrome will use the system-wide certificates (such as the ones installed above) or its own certificates. If you are having trouble getting a system-wide certificate to work for your internal website, you may wish to try directly installing a root certificate to Chrome (or any Chromium browser) directly.

[L246] 

[L247] To begin, open the **Settings** tab in the Chrome browser.

[L248] 

[L249] ![](https://support.optisigns.com/hc/article_attachments/35184720118803)

[L250] 

[L251] Next, click **Privacy and security** **→ Security → Manage Certificates**

[L252] 

[L253] **![](https://support.optisigns.com/hc/article_attachments/35184705392531)**

[L254] 

[L255] This will open the Certificates manager. You’ll need to add your internal website certificate as an authority. Here, click the **Trusted Root Certification Authorities** tab, then click **Import**.

[L256] 

[L257] ![](https://support.optisigns.com/hc/article_attachments/35184720125715)

[L258] 

[L259] This will open the Certificate Import Wizard. Click **Browse** and locate the certificate necessary for your internal website. Then, click **Next**.

[L260] 

[L261] **![](https://support.optisigns.com/hc/article_attachments/35184720127635)**

[L262] 

[L263] On the next screen, **Place all certificates in the following store** and make sure it’s “Trusted Root Certification Authorities”. Then hit **Next**.

[L264] 

[L265] **![](https://support.optisigns.com/hc/article_attachments/35184705364115)**

[L266] 

[L267] Now you’ll be asked to confirm whether you want to complete the import. As long as you trust the certificate, agree to it by hitting **Finish**.

[L268] 

[L269] **![](https://support.optisigns.com/hc/article_attachments/35184705368339)**

[L270] 

[L271] Congratulations, you’ve installed your root certificate on your Chrome browser.

[L275] ## Setting an Internal Website Up for Use on OptiSigns Using the Website App

[L276] 

[L277] The OptiSigns Website app allows you to display an internal website on your screens once a valid private certificate (.pem file) has been installed on your device.

[L278] 

[L279] To set it up, navigate to **Files/Assets** in your OptiSigns portal. Find the **Website** app.

[L280] 

[L281] ![](https://support.optisigns.com/hc/article_attachments/35184705404947)

[L282] 

[L283] Enter the **URL** of your webpage here. The Name field is irrelevant to the app’s function and will only make the asset easier to find in OptiSigns.

[L284] 

[L285]   | **NOTE** |

[L286] | If your internal website is large and complex, you can also display it using the **Advanced Website** app. If you’re having trouble displaying it on the basic Website app, create an Advanced Website asset and try again. |

[L287] 

[L288] Once the Website asset is created, it can be assigned to a screen.

[L289] 

[L290] ### Using Web Scripting to Bypass Logins

[L291] 

[L292] Many internal websites require logins to work. In this case, the Website app will not be sufficient to access them on the OptiSigns app. In this case, we recommend our Web Scripting app.

[L293] 

[L294] Follow our complete [guide to using the Web Scripting app](https://support.optisigns.com/hc/en-us/articles/1500012522362-How-to-use-the-Web-Scripting-App) to set that up.
