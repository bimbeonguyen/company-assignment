[L1] # Tutorial: Creating Schedules and Adding Schedule Items Using GraphQL

[L2] 

[L3] Using GraphQL, it’s possible to create a schedule on OptiSigns, then add items to it and assign values to them. We’ll cover each of these steps in turn.

[L4] 

[L5] **1. Creating a New Schedule**

[L6] 

[L7] To create a new schedule, we’ll need a specific Mutation:

[L8] 

[L9] ```

[L10] mutation CreateSchedule($payload: ScheduleInput!, $teamId: String)  

[L11]   

[L12] {saveSchedule (payload:$payload,teamId:$teamId){  

[L13]    _id,  

[L14]    accountId,  

[L15]    createdAt,  

[L16]    createdBy,  

[L17]    groupId,  

[L18]    lastTeamId,  

[L19]    lastUpdatedBy,  

[L20]    lastUpdatedDate,  

[L21]    name,  

[L22]    path,  

[L23]    tags,  

[L24]    teamId  

[L25]    }  

[L26] }

[L27] ```

[L28] 

[L29] **Variables:**

[L30] 

[L31] ```

[L32] {“payload”: {  

[L33]    “groupId”:  

[L34]    “Name”:  

[L35]    “Path”:  

[L36]    “Tags”:  

[L37]    “teamId”:  

[L38]    }  

[L39] }

[L40] ```

[L41] 

[L42] ![](https://support.optisigns.com/hc/article_attachments/36558841524243)

[L43] 

[L44] This Mutation creates a brand new schedule, as you can see from the data to the right:

[L58] **3. Adding Assets or Playlists to the Schedule**

[L59] 

[L60] Now, we’ll want to add an item to the schedule. For that, we’ll need another Mutation:

[L61] 

[L62] ```

[L63] mutation addScheduleItem($force:Boolean,$payload: AddScheduleItemInput!, $teamId: String!){  

[L64]   

[L65]   addScheduleItem(force:$force,payload:$payload,teamId:$teamId){  

[L66]   

[L67]          _id,  

[L68]   

[L69]        name,  

[L70]   

[L71]        assetId,        

[L72]   

[L73]        teamId,  

[L74]   

[L75]        playlistId,  

[L76]   

[L77]        repeatObject{  

[L78]   

[L79]          id,  

[L80]   

[L81]          repeat,  

[L82]   

[L83]          text,  

[L84]   

[L85]          type,  

[L86]   

[L87]          rrule  

[L88]   

[L89]        },  

[L90]   

[L91]        range{startDate,endDate},  

[L92]   

[L93]        documentDuration  

[L94]   

[L95]   }  

[L96]   

[L97] }

[L98] ```

[L99] 

[L100] ![](https://support.optisigns.com/hc/article_attachments/36558841535507)

[L101] 

[L102] **Variables:**

[L103] 

[L104] ```

[L105] {"force": false,  

[L106]  "payload": {"scheduleId": "",  

[L107]             "assetId": "",  

[L108]             "playlistId":"",  

[L109]             "type": "",  

[L110]             "repeatObject": {  

[L111]               "rrule": ""  

[L112]             },  

[L113]              "range": {  

[L114]                "startDate": "",  

[L115]                "endDate": ""  

[L116]             }  

[L117]      },  

[L118]  "teamId": ""  

[L119] }

[L120] ```

[L128] **4. Editing Schedule Items**

[L129] 

[L130] Once you’ve created a schedule and set an item on it, it’s possible to edit those items. This requires yet another Mutation:

[L131] 

[L132] ```

[L133] mutation updateScheduleItem($force:Boolean,$payload: UpdateScheduleItemInput!, $scope: APPLY_SCHEDULE_ITEM_SCOPES!){  

[L134]   

[L135]   updateScheduleItem(force:$force,payload:$payload,scope:$scope){  

[L136]        _id,  

[L137]        name,  

[L138]        assetId,        

[L139]        teamId,  

[L140]        playlistId,  

[L141]        repeatObject{  

[L142]          id,  

[L143]          repeat,  

[L144]          text,  

[L145]          type,  

[L146]          rrule  

[L147]        },  

[L148]        range{startDate,endDate},  

[L149]        documentDuration  

[L150]   }  

[L151] }

[L152] ```

[L153] 

[L154] ![](https://support.optisigns.com/hc/article_attachments/36558834989843)

[L155] 

[L156] With this mutation, we’ll input these variables:

[L157] 

[L158] ```

[L159] {"force": false,  

[L160]  "payload": {"_id": "",  

[L161]             "assetId": "",  

[L162]             "playlistId":"",  

[L163]             "repeatObject": {  

[L164]               "rrule": ""  

[L165]              },  

[L166]             "range": {  

[L167]               "startDate": "",  

[L168]               "endDate": ""  

[L169]             }  

[L170]  },  

[L171]  "scope": ""  

[L172] }

[L173] ```

[L174] 

[L175] ![](https://support.optisigns.com/hc/article_attachments/36558834991379)

[L176] 

[L177] Here, we’ve changed the repeat frequency from daily to weekly, and the start date from December 18, 2024 to December 20, 2024. This is reflected in the data retrieved:

[L178] 

[L179] ![](https://support.optisigns.com/hc/article_attachments/36558841549075)
