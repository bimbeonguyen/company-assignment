[L1] # Get Started with OptiSigns API

[L2] 

[L3] The easiest way to get started with the development is to use the provided web-based GraphQL IDE. ¬†You can access it from here.  

[L4] <https://graphql-gateway.optisigns.com/graphql>¬†

[L5] 

[L6] You also will need API key to provide authorization to your account. You can learn how to get your API key from¬†[here](https://support.optisigns.com/hc/en-us/articles/4414563797139-Generate-Manage-OptiSigns-API-Key).

[L7] 

[L8] #### **Query and Mutation**

[L9] 

[L10] GraphQL provides two ways than you can interact with your data. Query is used to read or fetch data from the server. Mutation is used to modify or post data to the server.

[L11] 

[L12] #### **Fetch the devices**

[L13] 

[L14] This query will list all the devices available in your account. You can specify what fields you are interested and get it returned from the result.

[L15] 

[L16] ```

[L17] query{  

[L18]   devices (query : {}) {  

[L19]     page{  

[L20]       edges{  

[L21]         cursor,  

[L22]         node{  

[L23]           _id,  

[L24]           deviceName,  

[L25]           UUID,  

[L26]           pairingCode,  

[L27]           currentType,  

[L28]           currentAssetId,  

[L29]           currentPlaylistId,  

[L30]           localAppVersion,  

[L31]         }  

[L32]       }  

[L33]     }  

[L34]   }  

[L35] }

[L36] ```

[L37]

[L40] #### **Fetch a specific asset**

[L41] 

[L42] If you want to fetch the data following a condition, you can do so by supply an argument. This query will get the asset named "Houston Weather Test" in your account.¬†

[L43] 

[L44] ```

[L45] query{  

[L46]   assets (query : {originalFileName:"Houston Weather Test"}) {  

[L47]     page{  

[L48]       edges{  

[L49]         cursor,  

[L50]         node{  

[L51]           _id,  

[L52]           appType,  

[L53]           fileType,  

[L54]           name,  

[L55]           filename  

[L56]         }  

[L57]       }  

[L58]     }  

[L59]   }  

[L60] }

[L61] ```
