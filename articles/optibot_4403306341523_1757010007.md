[L1] # How to set up Monitoring Alert feature 

[L2] 

[L3] Monitoring & Alert is a Pro Plus and Enterprise feature that will allow you to receive notifications when devices are down.

[L4] 

[L5] **You will be able to set up:**

[L6] 

[L7] * Which devices to monitor, maybe not all devices are important (you can choose to receive devices with certain tags only i.e. [important screens])

[L8] * How many minutes or hours of downtime before notification sent

[L9] * To whom the notification should be sent (list of emails)

[L10] * You can create as many rules as you like (i.e. Location A screens down will send email to admin in Location A, Location B screens down will send email to admin in location B, etc.)

[L11] 

[L12] **Here are some best practices for you to get the most out of this feature and your screens:**

[L13] 

[L14] * Set the right expectation: if you have a lot of screens, it is probably not practical to set a goal of 100% screen up 24/7, for objective reasons like network, power, or hardware failure there may be some devices down for some time.

[L15] * If you expect your devices to run for X hours/day, you can run the [Proof of Play report](https://support.optisigns.com/hc/en-us/articles/360058936513) by devices and see how many hours your devices have run over last 7-30 days and see which devices are not meeting the X hours/day goal

[L16] * Set up a process, and training in place for the location's admin to help you troubleshoot when there are issues.

[L17] * If you are not running your screen 24/7, say your business hour is 7 AM - 7 PM, set an alert if downtime is >13 hours so you will not have a notification every day, and still be able to catch the screen down within 24 hours.

[L18] 

[L19] #### **How to set up the Monitoring Alert feature:**

[L20] 

[L21] Go to Account Settings and click Monitoring Alert.  

[L22] Or you can go straight to this page: <https://app.optisigns.com/app/s/monitoring-alert-rules>

[L23] 

[L24] ![mceclip0.png](https://support.optisigns.com/hc/article_attachments/4403292193555)

[L25] 

[L26] Click Create New Policy:

[L27] 

[L28] ![mceclip1.png](https://support.optisigns.com/hc/article_attachments/4403298566675)

[L29] 

[L30] Fill in the settings for your policy:

[L31] 

[L32] ![mceclip2.png](https://support.optisigns.com/hc/article_attachments/4403306261651)

[L33] 

[L34] * Alert Policy Name: This is just a name to show in the list of policies since you can create more than one.

[L35] * Alert Rule: how often do you want monitoring rules to run? Our suggestion is 60 minutes. While OptiSigns support monitoring at 15-minute intervals, 60min may help you avoid some false positives due to temporary network, or power issues

[L36] * Send Reminder: how often do you want OptiSigns to send reminders when screens are down? Our suggestion is to send reminders once a day. While OptiSigns support sending reminders every hour, this may just clutter up your inbox if you are not going to take action on the screen within 1h of getting a notification.

[L37] * Select Screens or Tags: select which group of screens you want to monitor, not all screens may be equally important to you. You can select group specifics screens or group of tags. In this example we select all screens tags as [Location-1] and [customer-facing]

[L38] * Send alert to: list of email address alerts to be sent to

[L39] * Time zone: which time zone you want to follow. In alert email will reference as of time, if device is in New York (EST), and you are in Houston (CST), the down time will be convert to Houston (CST) time. This is necessary and useful especially if you manage many devices across time zones.

[L40] * Active: you can turn the rule on/off. Sometimes you may want to turn it off temporarily if you are doing changes like building renovation or network upgrade that you know devices will not be on, so you don't receive too many alerts.
