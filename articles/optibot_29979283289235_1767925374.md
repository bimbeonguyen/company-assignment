[L104] ## How to Handle Empty Data Values in a Repeater

[L105] 

[L106] Let's say you're playing this design as a single asset or running it like a slideshow, but it is nobody's birthday. You can set how you would like your Repeater to handle empty data values.┬Ā

[L107] 

[L108] 1. In Designer, **select your Repeater** so it is highlighted.

[L109] 2. Select **Settings** from the top bar.

[L110] 3. In the far-right menu, use the drop-down menu for **Empty Data Handling** to select:

[L111] 	* **Skip:** Skip it, or get rid of additional repeater items if there isn't enough data to reach the set "Total Items displayed per Page".

[L112] 	* **Use Default Value:** Show default content, which is what your Repeater element looks like by itself.

[L113] 	* **Use Blank:** The Repeater will show nothing.

[L114] 4. Make sure to **save** your design.

[L120] ## Set Up Auto Sorting in Your Sheet

[L121] 

[L122] #### When handling large quantities of data in your spreadsheet, the ability to auto sort is incredibly useful and at times, essential. Instead of manually rearranging your data every time you add or update entries, the SORT function does it for you automatically.

[L123] 

[L124] |  |

[L125] | --- |

[L126] | **Note:** If you're using the example spreadsheet we provided above, there are specific instructions in that Google Sheet on how to set up auto-sorting for that data set-up. Please read the "**Tips**" section. If you're not using that spreadsheet, you can follow the example below to help you understand sorting.┬Ā |

[L127] 

[L128] #### **Example: Auto-Sorting a Birthday List**

[L129] 

[L130] Imagine you have a list of employees and their birthdates. You want to keep this list sorted by the upcoming birthdays, and if two employees share the same birthdate, you want to sort their names alphabetically.

[L131] 

[L132] LetŌĆÖs go over the **Step-by-Step Instructions:**

[L133] 

[L134] **Step 1**: Open Your Google Sheet. For simplicity, hereŌĆÖs how your data might look:

[L135] 

[L136] | **Celebrant Name** | **Birthdate** |

[L137] | Alice | 12/05/1990 |

[L138] | Bob | 12/15/2001 |

[L139] | Carol | 12/05/1990 |

[L140] | David | 12/01/1988 |

[L141] 

[L142] **Step 2:** Select an Empty Cell

[L143] 

[L144] Click on an empty cell where you want the sorted list to appear. For this example, weŌĆÖll use cell A1 in a new tab.

[L145] 

[L146] **Step 3**: Enter the SORT Formula

[L147] 

[L148] In the selected cell A1, enter the following formula and press Enter.

[L149] 

[L150] ```

[L151] ={"Celebrant Name", "Birthdate"; SORT(A2:B, B2:B, true, A2:A, true)}

[L152] ```

[L153] The formula will immediately sort the data and display the results starting from Cell A1.

[L154] 

[L155] ### How the Formula Works:

[L156] 

[L157] 1. **={"Celebrant Name", "Birthdate";**

[L158] 	* **Purpose**: This creates the headers for your sorted data.

[L159] 	* **Explanation**: It sets up the column titles "**Celebrant Name**" and "**Birthdate**" at the top of the sorted list. The **semicolon** **;** tells Google Sheets to display the sorted data below these headers.

[L160] 2. **SORT(A2:B, B2:B, true, A2:A, true)**

[L161] 	* **A2:B**: This is the range of data you want to sort (including both columns: Celebrant Name and Birthdate).

[L162] 	* **B2:B**: This specifies the first column to sort by, which is Birthdate.

[L163] 		1. **true**: This indicates that the sorting should be in ***ascending*** order.

[L164] 		2. **false**: This indicates that the sorting should be in ***descending*** order.

[L165] 	* **A2:A**: This specifies the second column to sort by, which is Celebrant Name.

[L166] 		1. **true**: This indicates that names should also be sorted in ***ascending*** order.

[L167] 		2. **false**: This indicates that names should also be sorted in ***descending*** order.

[L168] 

[L169] ### **Sorted Table:**

[L170] 

[L171] After applying the formula, your sorted list would look like this:

[L172] 

[L173] | **Celebrant Name** | **Birthdate** |

[L174] | David | 12/01/1988 |

[L175] | Alice | 12/05/1990 |

[L176] | Carol | 12/05/1990 |

[L177] | Bob | 12/15/2001 |

[L178] 

[L179] * David has the earliest birthday (12/01/1988) and appears first.

[L180] * Alice and Carol share the same birthdate (12/05/1990), so they are sorted alphabetically.

[L181] * Bob is listed last with the latest birthdate (12/15/2001).

[L182] 

[L183] Your list is now automatically sorted by birthdate and name.┬Ā

[L184] 

[L185] Any updates or additions to the original data in your birthday columns will trigger the list in your new tab to auto-sort accordingly.
