[L1] # How to use Toast API data with OptiSigns

[L2] 

[L3] ### Toast is one of the most common Point-of-Sale systems in North America. In this article, we'll explain how to set up API integration between your Toast system and OptiSigns.

[L4] 

[L5] * [Step 1: Preparation](#Step1)

[L6] * [Step 2: Authentication to Toast API](#Step2)

[L7] * [Step 3: Call the API to Get the Required Data from Toast](#Step3)

[L8] * [Step 4: Build the DMB Designer with OptiSync](#Step4)

[L9] 

[L10] Toast provides an API for users to integrate POS data with other systems. With OptiSync, building auto-updating digital menu boards from your Toast API's data is simple.

[L11] 

[L12] Toast API requires a dynamically generated authentication token to be supplied with every API call, unlike many other POS systems that use a static API key. This adds some complexity to the integration that other POS systems don't have. However, using OptiSigns' API request and OptiSync, these added complexities are no trouble at all.

[L13] 

[L14] In this article, we'll provide a step-by-step guide on how to integrate your Toast API using OptiSigns API request, and how to create a DMB using OptiSync. Below are the high level steps we will follow:

[L15] 

[L16] 1. Get required information from Toast portal

[L17] 

[L18] 2. Get the Authentication Token from Toast (Inside Pre-Request step)

[L19] 

[L20] 3. Call the API to get the required data from Toast

[L21] 

[L22] 4. Use Toast API data to build DMB in designer with OptiSync

[L23]

[L25] 

[L26] ## Step 1: Preparation

[L27] 

[L28] To get started with integrating a Toast API with OptiSigns, we need the following information from the Toast system:

[L29] 

[L30] * *toast-api-hostname*

[L31] * clientId

[L32] * clientSecret

[L33] * userAccessType

[L34] 

[L35] Refer to Toast's Documentation [here](https://doc.toasttab.com/doc/devguide/portalHowToBuildAToastIntegration.html) for further details.

[L36] 

[L37] |  |

[L38] | --- |

[L39] | **NOTE** |

[L44] ## Step 2: Authentication for Toast API

[L45] 

[L46] For Toast API authentication, we will first need to pass the POST request to get the authentication token. The authentication token is then used to pass in the API request to get the data from Toast menus, orders etc.┬Ā

[L47] 

[L48] This authentication process will be handled using Pre-request processing with OptiSigns' API request. For more information about Pre-request processing and API requests in general, please check [here](https://support.optisigns.com/hc/en-us/articles/22875592994195).

[L49] 

[L50] In the Pre-request processing stage, the OptiSigns API calls the authentication API to get the necessary token, and sets it to the context of the API request.

[L51] 

[L52] In this example, the token is set to the context variable "authorization". When the API request is made, it will be able to use the authentication token. Below is a screenshot of this example in practice.

[L53] 

[L54] ![](https://support.optisigns.com/hc/article_attachments/31870675893011)

[L55] 

[L56] Use this code snippet (with the data obtained earlier filling in the "xxx"s) to

[L57] 

[L58] ```

[L59] const body = {  

[L60] "clientId": "xxx",  

[L61] "clientSecret": "xxx",  

[L62] "userAccessType": "xxx"  

[L63] };  

[L64] const {data, headers} = await os.postRequest("https://[toast-api-hostname]/authentication/v1/authentication/login", body );  

[L65] const token = os.getValue(data.token.accessToken);  

[L66] os.context.set("authorization", os.getValue(token));

[L67] ```

[L68]

[L70] 

[L71] ## Step 3: Call the API to get the required data from Toast

[L72] 

[L73] Now we'll use the authorization token we received in Pre-request processing and pass it to the actual API call header.┬Ā

[L74] 

[L75] In the Header tab, create two parameters with the following values:

[L76] 

[L77] **authorization** Bearer {{authorization}}

[L78] 

[L79] **Toast-Restaurant-External-ID**┬Ā

[L80] 

[L81] You can get the **Toast-Restaurant-External-ID value** from Toast Portal. This is the specific restaurant Id you want to get data for.

[L82] 

[L83] ![](https://support.optisigns.com/hc/article_attachments/31870675894035)

[L84] 

[L85] Now put the desired API URL from which you want to get data. In this example we have used the following API to get the menus

[L86] 

[L87] * + <https://ws-api.toasttab.com/menus/v2/menus>

[L88] 

[L89] The final request will look something like this:

[L90] 

[L91] ![](https://support.optisigns.com/hc/article_attachments/31870675898515)

[L92] 

[L93] You can enable this request and save the API. Click **Run Test**.

[L94] 

[L95] You should receive a *(200 OK)*┬Āresponse, with data returning from the API. This means the API request has successfully contacted Toast and is transferring data.

[L96] 

[L97] ![](https://support.optisigns.com/hc/article_attachments/31870683910291)

[L100] 

[L101] ## Step 4: Build Digital Menu Board in Designer with OptiSync

[L102] 

[L103] Now that your API request data source is ready for use, you can build your DMB using Designer with OptiSync. OptiSync allows you to map the API data to an element in the designer, either text or images. Using Repeaters, this data can be used to build out multiple item menus, and you're also able to define how you'd like to handle sold-out items, specials, and more.

[L104] 

[L105] For a step-by-step guide and more examples, please see our article on [Building Digital Menu Boards Using OptiSync](https://support.optisigns.com/hc/en-us/articles/31860170199955-Integrating-Point-of-Sale-POS-Systems-to-Build-Digital-Menu-Boards-with-OptiSync).
