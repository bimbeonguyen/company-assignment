[L1] # How to set up SAML 2.0 with OptiSigns and OneLogin

[L2] 

[L3] With Pro Plus and Enterprise plan, you can configure SAML 2.0 with OptiSigns via OneLogin.

[L4] 

[L5] OneLogin will be acting as the IDP (Identify Provider), and OptiSigns will be working as the SP(Service Provider).

[L6] 

[L7] ┬Ā

[L8] 

[L9] ### **Set up OptiSigns & OneLogin:**

[L10] 

[L11] **First you need to do some set up in OptiSigns:**

[L12] 

[L13] If you don't have a sub domain yet, you can set up one by going to:  

[L14] <https://app.optisigns.com/app/s/branding-settings>

[L15] 

[L16] Fill in subdomain field and click Activate. After that you can use this sub domain for "  

[L17] You can also map your own domain like digitalsigns.yourcompany.com by following this [article](https://support.optisigns.com/hc/en-us/articles/1500000480302).

[L18] 

[L19] This will be the URL that you can share with your users so they can log in to use the app, once integration has set up. In our example we will use <https://advanced.optisigns.net/>

[L20] 

[L21] ![mceclip13.png](https://support.optisigns.com/hc/article_attachments/19053401939731)

[L22] 

[L23] Next go to SAML Single Sign On setting page:

[L24] 

[L25] <https://app.optisigns.com/app/s/saml-settings>

[L26] 

[L27] Click Enable SAML SSO.

[L28] 

[L29] The settings are:

[L30] 

[L31] * Enable Username & Password login: Allow users to also log in with username/password. ItŌĆÖs recommended to disable once integration is all done. As Admin/Owner, it's recommended that you keep at least 1 account with password log in, in case there's issues, you can always log back in from app.optisigns.com to reconfigure.

[L32] * Enable User Creation: If users are authenticated, but do not exist in OptiSigns, they will be created in OptiSigns. You should enable this, because you likely already assign/approve users/groups to use OptiSigns, unless for some reason you want to be very strict and want to review roles of users before they can start using OptiSigns.

[L33] * Enable User Override: Every time a user logs in, if their group assignment have changed on SAML, OptiSigns will update, override new profile settings.

[L34] * Note the "Single Sign On URL" and "Audience URI (SP Entity ID) URL", you will need this to use in OneLogin later.

[L35] 

[L36] ![](https://support.optisigns.com/hc/article_attachments/19053301304083)

[L37] 

[L38] ┬Ā

[L39] 

[L40] **Next, add OptiSigns as an App in your OneLogin admin portal:**

[L41] 

[L42] Log in to your OneLogin portal as admin -> Applications

[L43] 

[L44] Click Add app

[L45] 

[L46] ![mceclip0.png](https://support.optisigns.com/hc/article_attachments/4407386965907)

[L47] 

[L48] On the next page, search for "SAML" in the search box, and then select the "SAML Custom Connector (Advanced)".

[L49] 

[L50] ![mceclip1.png](https://support.optisigns.com/hc/article_attachments/4407387031315)

[L51] 

[L52] Enter "OptiSigns" in the display name, then click Save. You can also upload the OptiSigns logo here as well.

[L53] 

[L54] ![mceclip2.png](https://support.optisigns.com/hc/article_attachments/4407392839187)

[L55] 

[L56] After saving, go to the configuration page. This is where you should provide the Single Sign On URL, and SP Entity ID you get from your OptiSigns SAML SSO setting.

[L57] 

[L58] * SP Entity ID from OptiSigns SAML SSO setting should be put under **Audience (EntityID)** and **Login URL**.

[L59] * Single Sign On URL from OptiSigns SAML SSO setting should be put under **Recipient**, **ACS URL validator,** and **ACS URL**.

[L60] * Change the **SAML initiator** to the **Service Provider**

[L61] 

[L62] ![](https://support.optisigns.com/hc/article_attachments/19053399114899)

[L63] 

[L64] Then go to the SSO page. Get these 3 highlighted information, these need to be maintained in the OptiSigns SAML SSO settings. After clicking View Details of the certificate, you can find the encoded content of the certificate, this will be needed in the next step.

[L65] 

[L66] ![](https://support.optisigns.com/hc/article_attachments/19053348948883)

[L67] 

[L68] ┬Ā

[L69] 

[L70] Go back to your OptiSigns account maintain above mentioned 3 fields, and save it.

[L71] 

[L72] Put the SAML 2.0 endpoint from OneLogin under the SAML 2.0 Endpoint.

[L73] 

[L74] Put the Issuer URL from OneLogin under Identity Provider Issuer.

[L75] 

[L76] Put the content from base64 encoded x509 certificate under Public Certificate.

[L77] 

[L78] ![](https://support.optisigns.com/hc/article_attachments/19053330062483)

[L79] 

[L80] Now your login portal & integration are all setup.

[L81] 

[L82] ┬Ā

[L83] 

[L84] #### **Assign & map users, and groups from OneLogin to OptiSigns**

[L85] 

[L86] It's not required, but recommended to create groups of users to be assigned, and map to OptiSigns Roles, and Teams so they will automatically have the right role & group.

[L87] 

[L88] **IMPORTANT NOTE: If you don't configure this, all users will be assigned User Role & Default Team (screenshot see below)**

[L89] 

[L90] ┬Ā

[L91] 

[L92] To configure how OptiSigns should map the user groups to OptiSigns Roles by going to:┬Ā<https://app.optisigns.com/app/s/saml-settings>

[L93] 

[L94] Scroll to Advanced Settings and create a mapping.  

[L95] Group Name (roles assigned to the user from OneLogin), Role (role in OptiSigns) mapping.┬Ā  

[L96] ![mceclip9.png](https://support.optisigns.com/hc/article_attachments/4407387515155)

[L97] 

[L98] It's best practice to create a group specifically for OptiSigns with name prefix with OptiSigns- and map to OptiSigns like below:

[L99] 

[L100] * optisigns-admins (SAML group) -> OptiSigns role: Admin

[L101] * optisigns-users (SAML group) -> OptiSigns role: Users

[L102] * optisigns-custom-role (SAML group) -> OptiSigns custom role that you create

[L103] 

[L104] **How to handle Unmapped users/groups:**

[L105] 

[L106] You can map the "Unmapped users/group" to No Team (Disabled)

[L107] 

[L108] This way they will receive an error when trying to log in and will have to reach out to Admins to get correct teams, and roles assigned. This can be used as a safeguard, in case some users accidentally get assigned OptiSigns app but not the right groups.

[L109] 

[L110] ![mceclip3.png](https://support.optisigns.com/hc/article_attachments/19053410145299)

[L111] 

[L112] Note that if you map a SAML group to a Team and then delete the team, it will result in the new user being mapped to No Team and will have to contact you to be assigned to a team to use the app.

[L113] 

[L114] ┬Ā

[L115] 

[L116] Next, go to your OneLogin portal. Go to the parameters page of the OptiSigns application. This is where you maintain the mapping of the attributes.

[L117] 

[L118] Create new custom parameters here by clicking the + icon. Currently, OptiSigns support attributes mapping of first name, last name, and group. You can set the customer parameter name to the same default attribute name used on OptiSigns, and then assign it to the corresponding values from OneLogin.

[L119] 

[L120] These mappings will pass information to OptiSigns on the user's Name and Group.

[L121] 

[L122] ![mceclip7.png](https://support.optisigns.com/hc/article_attachments/4407387440147)

[L123] 

[L124] ┬Ā

[L125] 

[L126] The parameter names are corresponding to OptiSigns

[L127] 

[L128] <https://app.optisigns.com/app/s/saml-settings>

[L129] 

[L130] OptiSigns accept firstName, lastName, and group by default. Instead of setting the parameter names to the default attribute name used on OptiSigns,┬Ā you can also change the attribute name on OptiSigns to match the parameter names you defined on OneLogin as well.

[L132] ![mceclip2.png](https://support.optisigns.com/hc/article_attachments/19053401940243)

[L138] ### **That's it!**

[L139] 

[L140] You have configured SAML 2.0 for OptiSigns with OneLogin.

[L141] 

[L142] Now your users can log in using the subdomain that you configured (in this case it was <https://advanced.optisigns.net/signIn>).

[L143] 

[L144] You can share the URL with your users and they can log in with their SSO credentials.
