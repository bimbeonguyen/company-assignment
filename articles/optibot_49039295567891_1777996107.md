[L1] # ChromeOS PWA Instructions

[L2] 

[L3] ### In this article, we will show how to set up OptiSigns on ChromeOS devices. This can be used for mass provisioning across numerous Chrome boxes without the use of any Chrome apps.

[L13] Chrome boxes are common for large-scale digital signage deployment as there are industrial-grade chrome boxes such as [AOpen Commercial Chromebox](https://amzn.to/33Zcu22) or [ASUS Chromebit](https://amzn.to/2FS8FDP).

[L14] 

[L15] Chrome devices are very stable, and provide excellent performance, they can also be managed by Google Chrome Device Management which will simplify your large-scale deployment, ensure enterprise security, and reduce device management overhead. You can learn more about Google's Chrome Device Management [here](https://support.google.com/chrome/a/answer/1289314?hl=en#:~:text=Enforce%20policies%20and%20manage%20apps&text=You%20can%20make%20Wi%2DFi,Manage%20policies%20for%20Chrome%20devices.).

[L16] 

[L17] Displaying OptiSigns on a ChromeOS device consists of these broad steps:

[L18] 

[L19] 1. Adding OptiSigns Player to your Google Admin account

[L20] 2. Adding the OptiSigns Player app as an extension

[L21] 3. Mass Provisioning devices (optional)

[L22] 4. Pushing Content to Your ChromeOS Device

[L27] 

[L28] * An OptiSigns [**Standard plan or higher**](https://www.optisigns.com/pricing)**\***

[L81] ## Mass Provisioning on ChromeOS Devices (optional)

[L82] 

[L83] Mass provisioning requires two steps:

[L84] 

[L85] 1. Creating a mass provisioning template on the OptiSigns Portal

[L86] 2. Adding the JSON of that template into your Policy for Extensions area in Google Admin

[L87] 

[L88] Let's do it.

[L89] 

[L90] ### Creating a Mass Provisioning Template on OptiSigns

[L91] 

[L92] To create a provisioning template, go to the OptiSigns portal and go to the **Username** ŌåÆ **More** ŌåÆ **Provisioning Templates**.

[L93] 

[L94] ![provisioning templates](https://support.optisigns.com/hc/article_attachments/50218184695315)

[L95] 

[L96] Hit **Create New Provisioning Template:**

[L97] 

[L98] ![optisigns create provisioning template](https://support.optisigns.com/hc/article_attachments/49041868805267)

[L99] 

[L100] You'll be asked to set up the template.

[L101] 

[L102] ![provisioning template creation window](https://support.optisigns.com/hc/article_attachments/49039280054419)

[L103] 

[L104] * **Template Name**: Name of your template, this is for you to distinguish it when you have multiple provisioning templates.

[L105] * **Device Name Prefix**: This is used to generate the device name during provisioning.

[L106] * **Device Name Suffix**: This is used to generate the device name during provisioning, the default setting will add timestamps as a suffix.

[L107] * **Folder**: The folder you want to have the provisioned devices to be created.

[L108] * **WIFI**: Select from the stored WIFI, need to be created first. Only required if you want to setup WIFI during provisioning. WIFI setup is normally not needed for ChromeOS deployment. Because the deployment will be managed through Chrome Device Management.

[L109] * **Time Zone**: Specify the time zone of the device.

[L110] * **Tags**: Specify the tags you want to associate to the devices.

[L111] * **Initial Default Content Type**: Used to set the initial content on the device after provisioning.

[L112] * **Orientation**: Set the orientation, landscape is the default.

[L113] * **Sync Play**: Used to set the turn on/off of the sync play feature. For more details of Sync Play feature, please click [here](https://support.optisigns.com/hc/en-us/articles/4412065189267-Synchronized-playback-Sync-Play-feature).

[L114] * **Location**: Set the location of the device.

[L115] 

[L116] #### Advanced

[L117] 

[L118] ![advanced creation window](https://support.optisigns.com/hc/article_attachments/49039280055571)

[L119] 

[L120] * **If device already paired**: Provides options for devices which have already been paired in OptiSigns and how to handle that:

[L121] 

[L122] ![wifi options](https://support.optisigns.com/hc/article_attachments/49039295547411)

[L123] 

[L124] Once the template is created, it will be available under the list of provisioning templates. You can download the config file and it will be available for deployment. Click **Download**, then **ChromeOS**.

[L125] 

[L126] ![adding chromeos to provisioning template](https://support.optisigns.com/hc/article_attachments/49039295552531)

[L127] 

[L128] Your configuration file will be "provisionting-template-<Your Template Name>.txt".

[L129] 

[L130] This file contains a JSON object. This is what you will need to set up auto-provisioning in the next step.

[L131] 

[L132] ![](https://support.optisigns.com/hc/article_attachments/49039295553299)

[L154] ## Adding OptiSigns Player to a Private Account

[L155] 

[L156] If you don't have a Google Admin account, you can still get the OptiSigns PWA to work on ChromeOS devices. Just follow these steps.

[L157] 

[L158] First, go to the Chrome Web Store and **Search extensions and themes:**

[L159] 

[L160] ![](https://support.optisigns.com/hc/article_attachments/50288483048467)Search **OptiSigns Digital Signage** and select it.

[L161] 

[L162] ![](https://support.optisigns.com/hc/article_attachments/50288515530899)Click **Add to Chrome** and wait for it to install.

[L163] 

[L164] ![](https://support.optisigns.com/hc/article_attachments/50288483061523)You can go to **My Extensions** to see if it successfully downloaded:

[L165] 

[L166] ![](https://support.optisigns.com/hc/article_attachments/50288515539091)Now you can launch OptiSigns from the ChromeOS launcher on any ChromeOS device (TV, etc.):

[L167] 

[L168] ![](https://support.optisigns.com/hc/article_attachments/50288483065875)

[L169] 

[L170] This will allow OptiSigns to run!
