[L36] ### Page Animations

[L37] 

[L38] **Page Animations** will animate in the order it is placed on the list. These also will animate only once, whenever the page loads.

[L39] 

[L40] For example, letŌĆÖs say we want the water on our example template to fly in from the right - to give the appearance of splashing the water bottles.

[L46] **Animation Types:**

[L47] 

[L48] * **Fade in**

[L49] * **Fade out**

[L50] * **Fly in from left**

[L51] * **Fly in from right**

[L52] * **Fly in from bottom**

[L54] * **Fly out to left**

[L60] * **Bounce X** - Bounce the element horizontally

[L61] * **Bounce Y** - Bounce the element vertically

[L62] * **Flash**

[L63] * **Zoom** - Alternates between zooming the element in and out

[L66] 

[L67] While we could provide detailed descriptions of each of these, itŌĆÖs more fun to play around with them and see how they fit your vision for yourself!

[L68] 

[L69] **Animation Timing:**

[L70] 

[L71] * **After previous** - Sets the animation to trigger after the previous animation on the list.

[L72] * **With previous** - Sets the animation to trigger at the same time as the previous animation.

[L73] * **From beginning** - Sets the animation to trigger as soon as the asset loads.

[L74] 

[L75] **Delay (sec)** - How long of a delay between the asset loading/previous animation and the animation going off.

[L76] 

[L77] **Animation Speed** - Changes how quickly the animation moves.

[L78]

[L81] | **NOTE** |

[L82] | To properly utilize Animation Timing, you will need to coordinate your choice with the Delay option.

[L83] For example, say we have three page animations, **A, B, and C**.* A will go first, right when the page loads. No delay is needed.

[L84] * B is set to "After previous" in order to follow A. Setting a Delay of 1s will cause B to animate 1s ***after*** A animation ends.

[L85] * C we want to fire at the same time as B. We select "With previous."

[L86] 	+ With no delay, this will cause it to fire right as the B animation ends, causing it to look wrong.

[L87] 	+ To fix this, we will need to set a delay of 1s (assuming Animation Speed is also 1). This will cause C to fire 1s ***before*** the B animation ends, timing it properly.

[L102] 

[L103] **Object Animations** will animate simultaneously, regardless of the order they are placed in. These can also be set up to animate indefinitely (or as long as the asset is shown on the screen).

[L104] 

[L105] For example, letŌĆÖs say we want our screen to stay up for quite some time (over a minute, say). For that, weŌĆÖll want to include some additional animation to make sure it doesnŌĆÖt get stagnant, and to continue to draw peopleŌĆÖs attention.

[L106] 

[L107] To do this, weŌĆÖll draw attention to the Hydration text element by creating an Object Animation.

[L108] 

[L109] ![](https://support.optisigns.com/hc/article_attachments/42305104955795)

[L110] 

[L111] The Object Animations tab is similar to the Page Animations tab, with a few key differences:

[L112] 

[L113] **Animation Timing**

[L114] 

[L115] * **Delay** - refers to the delay **between** each animation. This includes at the beginning, and on every subsequent animation

[L116] * **End Delay** - Refers to the delay **inside** each animation.

[L117] 

[L118] **Loop**

[L119] 

[L120] * **Indefinitely** - Continually loop the animation until the asset is changed.

[L121] * **One Time** - Play the animation upon load in one time.

[L122] * **Iterations** - Allows you to specify how many times youŌĆÖd like the animation to repeat.

[L147] ## Common Issues with Animations

[L148] 

[L149] Now, letŌĆÖs go over a few common issues you may run into while trying to display your animations on a screen.

[L150] 

[L151] ### Preloading Assets in Playlist

[L152] 

[L153] This feature allows a more seamless appearance by loading assets prior to showing onscreen. This lessens the likelihood of your signage hanging for a second or two as the assets are loaded.

[L154] 

[L155] OptiSigns will preload your assets 6-8 seconds prior to the asset displaying on the screen. This has the unfortunate effect of potentially skipping your animation, which will display immediately upon loading. To fix this, you can either:

[L156] 

[L157] * ***(Recommended**)* Add a 1-3 second delay on your animations to ensure they appear onscreen when this preloading is selected:

[L158] * Disable Preload Assets in Playlist. However, this will show a blank screen in-between transitions.

[L159] 

[L160] ### Device Performance

[L161] 

[L162] Running too many animations at once will affect your deviceŌĆÖs performance and create a less-than-ideal digital signage experience.

[L163] 

[L164] In general, the degree to which this is felt will depend on your deviceŌĆÖs performance capabilities. A Roku device, typical smart TV, Amazon FireStick, or other device not purpose built for digital signage is likely to suffer from worse issues. If using one of these devices, we recommend no more than **1 or 2 animations per created asset**.

[L165] 

[L166] By contrast, Windows and Linux devices or OptiSigns players can handle significantly higher loads. However, if you see stuttering while using one of these devices, we recommend limiting animations to **3-4 animation per created asset**.
