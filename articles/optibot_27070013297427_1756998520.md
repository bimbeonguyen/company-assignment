[L1] # Display Device Info Onscreen

[L2] 

[L3] ### In this article, we will guide you through the process for displaying your device's information on screen.

[L4] 

[L5] * [Before You Begin](#Begin)

[L6] * [Step 1: Create Designer Asset](#Step1)

[L7] * [Step 2: Add a Device Data DataSource to the Asset and Map It](#Step2)

[L8] 	+ [Optional: Creating Additional Device Attributes](#Optional)

[L9] * [Step 3: Assign the Asset to Screens](#Step3)

[L10] 

[L11] Displaying your device's information onscreen has various advantages. Information like the screen name, assigned contents, and more can be helpful when:

[L12] 

[L13] * Managing a large network.

[L14] * Many screens are placed together in the same location.

[L15] * Running deployment or troubleshooting.

[L16] 

[L17] Device information can be easily displayed using the OptiSigns Dynamic Data Mapping feature. By using this, you'll only need to create a single asset, which can be used on any device to display that specific device's information. Let's get started.

[L18] 

[L19] ---

[L20] 

[L21] ## Before You Begin

[L22] 

[L23] Dynamic Data Mapping is available to **Pro Plus** subscribers or above.

[L24] 

[L25] Setting up a dynamic Device Info asset consists of three steps:

[L26] 

[L27] 1. Create a designer asset that fits your needs

[L28] 2. Enable the data source for device info and maintain the mapping.

[L29] 3. Assign the asset to your screen.

[L30] 

[L31] ---

[L32] 

[L33] ## Step 1: Create a Designer Asset

[L34] 

[L35] To begin, go to the **Files/Assets** tab in the OptiSigns web portal. Click **Apps**.

[L36] 

[L37] ![firefox_k4Purluovg.jpg](https://support.optisigns.com/hc/article_attachments/38883675197587)

[L38] 

[L39] Now, open the **Designer** app:

[L40] 

[L41] ![firefox_Bo3ccQ4spb.jpg](https://support.optisigns.com/hc/article_attachments/38883666973459)

[L42] 

[L43] Once you've entered the Designer app, you'll need to create your Designer asset. This can display any way you'd like. If you wish to reuse it and apply it to all your screens, you may need to make this asset a generic size for placing into a [**Split Screen zone**](https://support.optisigns.com/hc/en-us/articles/360026559573-How-to-Create-and-Use-the-Split-Screen-App).

[L44] 

[L45] In this example, we will set the canvas size to **585 x 315** in order to best fit a Split Screen zone. We'll create some placeholder text here to replace with our dynamic data in the next step:

[L46] 

[L47] ![](https://support.optisigns.com/hc/article_attachments/42859076732691)

[L48]

[L50] 

[L51] ## Step 2: Add a Device Metadata DataSource to the Asset and Map It

[L52] 

[L53] Next, we'll need to add a Device Metadata DataSource to retrieve device data. Click the **DataSource** tab on the left side of the screen, then hit **Add DataSource** at the top of the menu:

[L54] 

[L55] ![](https://support.optisigns.com/hc/article_attachments/42859068384403)

[L56] 

[L57] A screen with numerous options will pop up. Scroll down until you see **Device Metadata** under Adv. DataSources, then click it.

[L58] 

[L59] ![](https://support.optisigns.com/hc/article_attachments/42859076737427)

[L60] 

[L61] You'll be taken back to the DataSource menu, but there will be a new option - **Device Data (default)**. Click it.

[L62] 

[L63] ![](https://support.optisigns.com/hc/article_attachments/42859068388243)

[L64] 

[L65] You'll be presented with data that looks something like this:

[L66] 

[L67] ![](https://support.optisigns.com/hc/article_attachments/42859068390675)

[L68] 

[L69] By mapping these values to your asset, the specific screen data will be pulled, and the values in red will be replaced with values ***specific to your screen***. In order to map them, simply click and drag the value into the field where you want them to display.

[L70] 

[L71] You can also display custom device attributes, but this is not necessary. If the default data is enough for you, skip to [**step 3**](#Step3).

[L72] 

[L73] ### Optional: Setting Device Attributes

[L74] 

[L75] In some cases, you may wish to configure a custom attribute on your device and have that displayed.

[L76] 

[L77] In the example above, we've made an "assetId" field, which is a custom attribute defined on the screen level. These attributes can denote anything - from the asset being displayed to the shop the device is used in. It's all up to you.

[L78] 

[L79] To create this, navigate to your **Screens** tab. Find the screen whose attributes you wish to change, then hit **Edit**. You'll be taken to the **Edit Screen** menu. Hit **Advanced ŌåÆ** **More ŌåÆ Device additional attributes**.

[L80] 

[L81] ![firefox_hdXgPSnyE2.jpg](https://support.optisigns.com/hc/article_attachments/38883666992659)

[L82] 

[L83] ![firefox_eigWT4aT3p.jpg](https://support.optisigns.com/hc/article_attachments/38883666993555)

[L84] 

[L85] This will bring up the **Device additional attributes** screen. Here, to create a new attribute, click **New Attribute:**

[L86] 

[L87] ![firefox_Z4jbp2DWQv.jpg](https://support.optisigns.com/hc/article_attachments/38883675214355)

[L88] 

[L89] Now, fill in the desired attributes with the appropriate data, then hit **Update**:

[L90] 

[L91] ![firefox_7ZsHFOeUuN.jpg](https://support.optisigns.com/hc/article_attachments/38883666999059)

[L92] 

[L93] These additional attributes will now map to a Device Metadata Datasource. You will need to configure these additional attributes per screen in order for them to display.

[L94] 

[L95] ---

[L96] 

[L97] ## Step 3: Assign the Asset to Screen

[L98] 

[L99] Once the mapping is complete, you can simply assign it to a screen. The asset will display information specific to the screen it's been assigned.

[L100] 

[L101] For the above example, let's assign it to a screen called "Demo Laptop". We've configured the "assetId" through Device Additional Attributes to be 123, and the rest of the data has been mapped appropriately. It should display like so:

[L102] 

[L103] ![mceclip0(1).jpg](https://support.optisigns.com/hc/article_attachments/38884919962643)

[L104] 

[L105] If we were to assign it to another screen, these values would change. This makes the Device Info asset highly valuable as a troubleshooting tool.

[L106] 

[L107] **That's all!**

[L108] 

[L109] This is how you can display device info on the screen using device data mapping feature.
